# 보안 정책 (Security Policy)

OpenClaw에서 보안 문제를 발견하셨다면, 즉시 비공개로 보고해 주시기 바랍니다.

## 보안 취약점 보고 방법
발견된 이슈가 해당하는 각 저장소로 직접 보고해 주세요:
- **Core CLI 및 게이트웨이**: [openclaw/openclaw](https://github.com/openclaw/openclaw)
- **macOS 데스크탑 앱**: `apps/macos` 디렉토리 내 보고
- **iOS/Android 앱**: 각 앱 디렉토리 내 보고
- **그 외**: 어디에 보고해야 할지 확실하지 않은 경우 **[security@openclaw.ai](mailto:security@openclaw.ai)**로 이메일을 보내주세요.

상세 보고 방법은 [신뢰 페이지(영문)](https://trust.openclaw.ai)를 참조하세요.

### 보고서 필수 항목
1. **제목**
2. **심각도 평가**
3. **영향도(Impact)**
4. **해당 구성 요소**
5. **재현 방법(Technical Reproduction)**
6. **환경 정보**
7. **해결 권장 사항**

*재현 단계나 해결 권장 사항이 없는 보고서는 우선순위에서 제외됩니다.*

## 운영자 신뢰 모델 (가장 중요)
OpenClaw는 한 대의 게이트웨이를 여러 사용자가 공유하는 '다중 사용자(Multi-tenant)' 환경을 기본 모델로 하지 않습니다.

- **인증된 게이트웨이 사용자는 프로젝트 운영자로 간주되며, 해당 인스턴스에 대해 신뢰된 권한을 가집니다.**
- 세션 ID나 레이블은 라우팅용이지, 사용자 간의 보안 경계가 아닙니다.
- **권장 모드**: 한 명의 사용자당 하나의 물리적 기기(또는 VPS)를 사용하고, 해당 기기에 하나의 게이트웨이를 설치하는 것입니다.
- 여러 명의 사용자가 OpenClaw를 사용해야 하는 경우, 각 사용자마다 별도의 VPS나 OS 계정을 할당하여 완전히 격리하는 것을 권장합니다.

## 보안 경계 제외 항목 (Out of Scope)
다음 항목들은 보안 취약점으로 간주되지 않거나 코드 수정이 이루어지지 않는 경우입니다:
- **프롬프트 인젝션(Prompt Injection)**: 정책/인증/샌드박스 경계를 우회하지 않는 단순한 프롬프트 조작 공격.
- **로컬 설정 수정**: 사용자가 직접 로컬 설정 파일(`~/.openclaw` 등)을 수정하여 발생하는 영향.
- **의도된 로컬 기능**: 운영자가 허용한 로컬 명령어 실행이나 로컬 파일 쓰기 기능 등.
- **샌드박스 미사용**: 샌드박스가 비활성화된 기본 상태에서 발생하는 호스트 사이드 실행 기능.

## 안전한 운영 가이드
- **로컬 연결 전용**: 게이트웨이는 기본적으로 로컬호스트(`127.0.0.1`)에만 바인딩하여 사용하세요.
- **공개 인터넷 노출 금지**: 게이트웨이를 공인 IP(`0.0.0.0`)에 직접 노출하지 마세요. 외부 접근이 필요한 경우 SSH 터널이나 Tailscale을 사용하는 것을 강력히 권장합니다.
- **신뢰 기반 플러그인**: 플러그인은 OpenClaw 프로세스와 동일한 권한으로 실행되므로, 신뢰할 수 있는 플러그인만 설치하세요.

## 실행 환경 요구 사항
### Node.js 버전
OpenClaw는 중요한 보안 패치가 포함된 **Node.js 22.12.0 버전 이상(LTS)**을 반드시 요구합니다.
