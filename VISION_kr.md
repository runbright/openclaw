## OpenClaw 비전

OpenClaw는 실제로 일을 수행하는 AI입니다.
당신의 기기에서, 당신의 채널에서, 당신의 규칙으로 실행됩니다.

이 문서는 프로젝트의 현재 상태와 방향을 설명합니다.
아직 초기 단계이므로 반복이 빠릅니다.
프로젝트 개요 및 개발자 문서: [`README.md`](README.md)
기여 가이드: [`CONTRIBUTING.md`](CONTRIBUTING.md)

OpenClaw는 AI를 배우고 진정으로 유용한 것을 만들기 위한 개인 실험장으로 시작했습니다:
실제 컴퓨터에서 실제 작업을 수행할 수 있는 어시스턴트.
여러 이름과 형태를 거쳐 발전했습니다: Warelay -> Clawdbot -> Moltbot -> OpenClaw.

목표: 사용하기 쉽고, 다양한 플랫폼을 지원하며, 프라이버시와 보안을 존중하는 개인 어시스턴트.

현재 집중 분야:

우선순위:

- 보안 및 안전한 기본값
- 버그 수정 및 안정성
- 설정 신뢰성 및 첫 실행 UX

다음 우선순위:

- 모든 주요 모델 제공자 지원
- 주요 메시징 채널 지원 개선 (및 수요가 높은 몇 가지 추가)
- 성능 및 테스트 인프라
- 더 나은 컴퓨터 사용 및 에이전트 하네스 기능
- CLI 및 웹 프론트엔드 전반의 사용성
- macOS, iOS, Android, Windows, Linux 컴패니언 앱

기여 규칙:

- PR 하나 = 이슈/주제 하나. 관련 없는 여러 수정/기능을 묶지 마세요.
- ~5,000줄 이상 변경된 PR은 예외적인 상황에서만 검토됩니다.
- 작은 PR을 한 번에 대량으로 열지 마세요. 각 PR에는 검토 비용이 있습니다.
- 매우 작은 관련 수정의 경우 하나의 집중된 PR로 묶는 것이 권장됩니다.

## 보안

OpenClaw에서 보안은 의도적인 트레이드오프입니다: 기능을 죽이지 않으면서 강력한 기본값.
목표는 위험한 경로를 명시적이고 운영자가 제어할 수 있게 만들면서 실제 작업에 강력하게 유지하는 것입니다.

정식 보안 정책 및 신고:

- [`SECURITY.md`](SECURITY.md)

우리는 안전한 기본값을 우선시하지만 신뢰된 고성능 워크플로우를 위한 명확한 조절 장치도 노출합니다.

## 플러그인 및 메모리

OpenClaw에는 광범위한 플러그인 API가 있습니다.
코어는 간결하게 유지하고 선택적 기능은 보통 플러그인으로 배포해야 합니다.

선호하는 플러그인 경로는 npm 패키지 배포와 개발을 위한 로컬 확장 로딩입니다.
플러그인을 만들면 자체 저장소에서 호스팅하고 유지관리하세요.
코어에 선택적 플러그인을 추가하는 기준은 의도적으로 높습니다.
플러그인 문서: [`docs/tools/plugin.md`](docs/tools/plugin.md)
커뮤니티 플러그인 목록 + PR 기준: https://docs.openclaw.ai/plugins/community

메모리는 한 번에 하나의 메모리 플러그인만 활성화할 수 있는 특수 플러그인 슬롯입니다.
현재 여러 메모리 옵션을 제공하며 시간이 지남에 따라 하나의 권장 기본 경로로 수렴할 계획입니다.

### Skills

기본 UX를 위해 일부 번들 스킬을 여전히 제공합니다.
새로운 스킬은 기본적으로 코어에 추가하지 말고 먼저 ClawHub(`clawhub.ai`)에 게시해야 합니다.
코어 스킬 추가는 드물어야 하며 강력한 제품 또는 보안상의 이유가 필요합니다.

### MCP 지원

OpenClaw는 `mcporter`를 통해 MCP를 지원합니다: https://github.com/steipete/mcporter

이를 통해 MCP 통합을 유연하게 유지하고 코어 런타임에서 분리합니다:

- 게이트웨이를 다시 시작하지 않고 MCP 서버를 추가하거나 변경
- 코어 도구/컨텍스트 표면을 간결하게 유지
- MCP 변경이 코어 안정성과 보안에 미치는 영향 감소

현재로서는 코어에 퍼스트클래스 MCP 런타임을 구축하는 것보다 이 브리지 모델을 선호합니다.
`mcporter`가 아직 지원하지 않는 MCP 서버나 기능이 있으면 해당 저장소에 이슈를 열어주세요.

### 설정

OpenClaw는 현재 설계상 터미널 우선입니다.
이를 통해 설정을 명시적으로 유지합니다: 사용자는 문서, 인증, 권한, 보안 상태를 미리 볼 수 있습니다.

장기적으로는 보안 강화가 성숙함에 따라 더 쉬운 온보딩 흐름을 원합니다.
사용자로부터 중요한 보안 결정을 숨기는 편의 래퍼는 원하지 않습니다.

### TypeScript를 선택한 이유

OpenClaw는 주로 오케스트레이션 시스템입니다: 프롬프트, 도구, 프로토콜, 통합.
TypeScript는 기본적으로 해킹 가능하도록 OpenClaw을 유지하기 위해 선택되었습니다.
널리 알려져 있고 반복이 빠르며 읽고, 수정하고, 확장하기 쉽습니다.

## 병합하지 않을 것 (현재)

- ClawHub에서 제공할 수 있는 새 코어 스킬
- 모든 문서에 대한 전체 번역 세트 (연기됨; 추후 AI 생성 번역 계획)
- 모델 제공자 카테고리에 명확히 맞지 않는 상업 서비스 통합
- 명확한 기능 또는 보안 격차 없이 이미 지원되는 채널 주변의 래퍼 채널
- `mcporter`가 이미 통합 경로를 제공하는 경우 코어의 퍼스트클래스 MCP 런타임
- 기본 아키텍처로서의 에이전트 계층 프레임워크 (관리자의 관리자 / 중첩 플래너 트리)
- 기존 에이전트 및 도구 인프라를 중복하는 무거운 오케스트레이션 레이어

이 목록은 로드맵 가드레일이지 물리학 법칙이 아닙니다.
강력한 사용자 수요와 강력한 기술적 근거가 이를 변경할 수 있습니다.
