# 저장소 가이드라인 (CLAUDE_kr.md)

이 문서는 OpenClaw 저장소의 관리 및 개발을 위한 가이드라인을 담고 있습니다.

## 저장소 구조 및 모듈 구성
- **소스 코드**: `src/` 디렉토리에 위치합니다.
  - `src/cli`: CLI 관련 코드
  - `src/commands`: 주요 명령어 로직
  - `src/infra`: 인프라 및 기반 코드
  - `src/media`: 미디어 파이프라인
- **테스트**: `*.test.ts` 파일이 소스 코드와 함께 위치합니다.
- **문서**: `docs/` 디렉토리에 위치합니다.
- **플러그인/확장**: `extensions/*` 디렉토리에 위치하며, 개별 `package.json`을 가집니다.

## 문서 작성 및 링크 (Mintlify)
- 문서는 Mintlify(docs.openclaw.ai)를 통해 호스팅됩니다.
- 내부 링크 작성 시: `.md` 확장자 없이 루트 상대 경로를 사용하세요 (예: `[설정](/configuration)`).
- 피터(Peter)가 링크를 요청할 때만 전체 URL(`https://docs.openclaw.ai/...`)을 사용하세요.
- 문서 제목에 대시(—)나 아포스트로피(')를 사용하지 마세요. (앵커 링크가 깨질 수 있습니다.)

## 빌드, 테스트 및 개발 명령어
- **기본 환경**: Node.js **22+** 버전이 필요합니다.
- **의존성 설치**: `pnpm install`
- **빌드**: `pnpm build`
- **린트 및 포맷 확인**: `pnpm check`
- **테스트 실행**: `pnpm test`
- **CLI 실행 (개발용)**: `pnpm openclaw ...`

## 코딩 스타일 및 명명 규칙
- **언어**: TypeScript (ESM)를 사용하며, `any` 사용을 지양하고 엄격한 타입을 권장합니다.
- **포맷팅**: Oxlint와 Oxfmt를 사용합니다. 커밋 전에 `pnpm check`를 실행하세요.
- **파일 크기**: 파일당 약 700줄(LOC) 이하로 유지하는 것을 권장하며, 필요시 도우미 함수를 추출하여 분리하세요.
- **제품명**: 문서 제목 등에서는 **OpenClaw**를 사용하고, CLI 명령어, 패키지명, 경로 등에서는 소문자 `openclaw`를 사용합니다.

## 커밋 및 Pull Request (PR) 지침
- 커밋 메시지는 간결하고 행동 중심적이어야 합니다 (예: `CLI: add verbose flag to send`).
- 하나의 PR에는 연관된 변경 사항만 그룹화하여 제출하세요.
- **AI 도움을 받은 PR**: AI 도구(Claude, Codex 등)를 사용하여 작성한 경우 PR 설명에 반드시 명시해 주세요.

## 보안 가이드라인
- 인증 정보나 비밀번호는 절대 저장소에 커밋하지 마세요.
- 보안 취약점을 발견한 경우 공개적인 Issue를 생성하지 말고, `SECURITY.md`에 따라 비공개로 보고해 주세요.
