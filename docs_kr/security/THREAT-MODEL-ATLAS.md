# OpenClaw 위협 모델 v1.0

## MITRE ATLAS 프레임워크

**버전:** 1.0-draft
**최종 업데이트:** 2026-02-04
**방법론:** MITRE ATLAS + 데이터 흐름 다이어그램
**프레임워크:** [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems)

### 프레임워크 귀속

이 위협 모델은 AI/ML 시스템에 대한 적대적 위협을 문서화하기 위한 산업 표준 프레임워크인 [MITRE ATLAS](https://atlas.mitre.org/)를 기반으로 합니다. ATLAS는 AI 보안 커뮤니티와 협력하여 [MITRE](https://www.mitre.org/)가 관리합니다.

**주요 ATLAS 리소스:**

- [ATLAS 기법](https://atlas.mitre.org/techniques/)
- [ATLAS 전술](https://atlas.mitre.org/tactics/)
- [ATLAS 사례 연구](https://atlas.mitre.org/studies/)
- [ATLAS GitHub](https://github.com/mitre-atlas/atlas-data)
- [ATLAS 기여](https://atlas.mitre.org/resources/contribute)

### 이 위협 모델에 기여하기

이것은 OpenClaw 커뮤니티가 관리하는 살아있는 문서입니다. 기여 가이드라인은 [CONTRIBUTING-THREAT-MODEL.md](./CONTRIBUTING-THREAT-MODEL.md)를 참조하세요:

- 새로운 위협 보고
- 기존 위협 업데이트
- 공격 체인 제안
- 완화 조치 제안

## 1. 소개

### 1.1 목적

이 위협 모델은 AI/ML 시스템을 위해 특별히 설계된 MITRE ATLAS 프레임워크를 사용하여 OpenClaw AI 에이전트 플랫폼과 ClawHub 스킬 마켓플레이스에 대한 적대적 위협을 문서화합니다.

### 1.2 범위

| 컴포넌트               | 포함 여부 | 참고                                              |
| ---------------------- | -------- | ------------------------------------------------- |
| OpenClaw Agent Runtime | 예       | 코어 에이전트 실행, 도구 호출, 세션               |
| Gateway                | 예       | 인증, 라우팅, 채널 통합                           |
| Channel Integrations   | 예       | WhatsApp, Telegram, Discord, Signal, Slack 등     |
| ClawHub Marketplace    | 예       | 스킬 게시, 관리, 배포                             |
| MCP Servers            | 예       | 외부 도구 제공자                                  |
| User Devices           | 부분     | 모바일 앱, 데스크톱 클라이언트                    |

### 1.3 범위 외

이 위협 모델에서 명시적으로 범위 외로 지정된 것은 없습니다.

## 4. ClawHub 공급망 분석

### 4.1 현재 보안 제어

| 제어              | 구현                        | 효과성                                              |
| ----------------- | --------------------------- | --------------------------------------------------- |
| GitHub 계정 연령  | `requireGitHubAccountAge()` | Medium - 새로운 공격자의 장벽을 높임                 |
| 경로 정화         | `sanitizePath()`            | High - 경로 순회 방지                                |
| 파일 유형 검증    | `isTextFile()`              | Medium - 텍스트 파일만, 그러나 여전히 악의적일 수 있음 |
| 크기 제한         | 50MB 전체 번들              | High - 리소스 고갈 방지                              |
| 필수 SKILL.md     | 필수 readme                 | Low 보안 가치 - 정보 제공용                          |
| 패턴 관리         | moderation.ts의 FLAG_RULES  | Low - 쉽게 우회 가능                                 |
| 관리 상태         | `moderationStatus` 필드     | Medium - 수동 검토 가능                              |

### 4.2 관리 플래그 패턴

`moderation.ts`의 현재 패턴:

```javascript
// 알려진 나쁜 식별자
/(keepcold131\/ClawdAuthenticatorTool|ClawdAuthenticatorTool)/i

// 의심스러운 키워드
/(malware|stealer|phish|phishing|keylogger)/i
/(api[-_ ]?key|token|password|private key|secret)/i
/(wallet|seed phrase|mnemonic|crypto)/i
/(discord\.gg|webhook|hooks\.slack)/i
/(curl[^\n]+\|\s*(sh|bash))/i
/(bit\.ly|tinyurl\.com|t\.co|goo\.gl|is\.gd)/i
```

**한계:**

- slug, displayName, summary, frontmatter, metadata, 파일 경로만 확인
- 실제 스킬 코드 내용은 분석하지 않음
- 단순 정규식은 난독화로 쉽게 우회 가능
- 행동 분석 없음

### 4.3 계획된 개선

| 개선                   | 상태                                  | 영향                                                                  |
| ---------------------- | ------------------------------------- | --------------------------------------------------------------------- |
| VirusTotal 통합        | 진행 중                               | High - Code Insight 행동 분석                                         |
| 커뮤니티 신고          | 부분 (`skillReports` 테이블 존재)     | Medium                                                                |
| 감사 로깅              | 부분 (`auditLogs` 테이블 존재)        | Medium                                                                |
| 배지 시스템            | 구현됨                                | Medium - `highlighted`, `official`, `deprecated`, `redactionApproved` |
