# OpenClaw 위협 모델 기여 가이드

OpenClaw를 더 안전하게 만드는 데 도움을 주셔서 감사합니다. 이 위협 모델은 살아있는 문서이며 누구나 기여를 환영합니다 - 보안 전문가일 필요는 없습니다.

## 기여 방법

### 위협 추가

우리가 다루지 않은 공격 벡터나 위험을 발견하셨나요? [openclaw/trust](https://github.com/openclaw/trust/issues)에 이슈를 열고 자신의 말로 설명해 주세요. 프레임워크를 알 필요도, 모든 필드를 채울 필요도 없습니다 - 시나리오만 설명해 주세요.

**포함하면 도움이 되는 내용 (필수는 아님):**

- 공격 시나리오와 악용 방법
- OpenClaw의 어떤 부분이 영향을 받는지 (CLI, gateway, channels, ClawHub, MCP servers 등)
- 심각도 판단 (low / medium / high / critical)
- 관련 연구, CVE 또는 실제 사례 링크

검토 중에 ATLAS 매핑, 위협 ID 및 위험 평가를 처리하겠습니다. 이러한 세부 사항을 포함하시면 좋지만, 필수는 아닙니다.

> **이것은 위협 모델에 추가하기 위한 것이며, 실제 취약점을 보고하기 위한 것이 아닙니다.** 악용 가능한 취약점을 발견하셨다면, 책임 있는 공개 지침은 [Trust 페이지](https://trust.openclaw.ai)를 참조하세요.

### 완화 조치 제안

기존 위협에 대한 해결 아이디어가 있으신가요? 위협을 참조하는 이슈나 PR을 열어주세요. 유용한 완화 조치는 구체적이고 실행 가능합니다 - 예를 들어, "rate limiting 구현"보다 "gateway에서 발신자당 분당 10개 메시지 rate limiting"이 더 좋습니다.

### 공격 체인 제안

공격 체인은 여러 위협이 결합되어 현실적인 공격 시나리오를 만드는 방법을 보여줍니다. 위험한 조합이 보인다면, 단계와 공격자가 어떻게 연결하는지 설명해 주세요. 공격이 실제로 어떻게 전개되는지에 대한 짧은 서술이 형식적인 템플릿보다 더 가치 있습니다.

### 기존 내용 수정 또는 개선

오타, 명확화, 오래된 정보, 더 나은 예시 - PR 환영합니다. 이슈 없이도 괜찮습니다.

## 우리가 사용하는 것

### MITRE ATLAS

이 위협 모델은 프롬프트 인젝션, 도구 오용, 에이전트 악용과 같은 AI/ML 위협을 위해 특별히 설계된 프레임워크인 [MITRE ATLAS](https://atlas.mitre.org/) (Adversarial Threat Landscape for AI Systems)를 기반으로 합니다. 기여하기 위해 ATLAS를 알 필요는 없습니다 - 검토 중에 제출물을 프레임워크에 매핑합니다.

### 위협 ID

각 위협은 `T-EXEC-003`과 같은 ID를 받습니다. 카테고리는 다음과 같습니다:

| 코드    | 카테고리                                 |
| ------- | ---------------------------------------- |
| RECON   | 정찰 - 정보 수집                         |
| ACCESS  | 초기 접근 - 진입 획득                    |
| EXEC    | 실행 - 악의적 행동 수행                  |
| PERSIST | 지속성 - 접근 유지                       |
| EVADE   | 방어 회피 - 탐지 회피                    |
| DISC    | 발견 - 환경 학습                         |
| EXFIL   | 유출 - 데이터 탈취                       |
| IMPACT  | 영향 - 피해 또는 중단                    |

ID는 검토 중에 유지관리자가 할당합니다. 직접 선택할 필요는 없습니다.

### 위험 수준

| 수준         | 의미                                                         |
| ------------ | ------------------------------------------------------------ |
| **Critical** | 전체 시스템 침해, 또는 높은 가능성 + 치명적 영향             |
| **High**     | 상당한 피해 가능성, 또는 중간 가능성 + 치명적 영향           |
| **Medium**   | 중간 위험, 또는 낮은 가능성 + 높은 영향                      |
| **Low**      | 가능성이 낮고 제한적인 영향                                  |

위험 수준이 확실하지 않으면, 영향만 설명해 주시면 우리가 평가하겠습니다.

## 검토 프로세스

1. **분류** - 48시간 이내에 새로운 제출물을 검토합니다
2. **평가** - 실현 가능성을 확인하고, ATLAS 매핑 및 위협 ID를 할당하고, 위험 수준을 검증합니다
3. **문서화** - 모든 것이 형식에 맞고 완전한지 확인합니다
4. **병합** - 위협 모델 및 시각화에 추가합니다
