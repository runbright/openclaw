# 세션 도구 (Session Tools)

OpenClaw는 에이전트가 다른 세션의 정보를 확인하고 상호작용할 수 있도록 몇 가지 전용 도구를 제공합니다. 이를 통해 에이전트는 과거 대화 내용을 참고하거나, 다른 대화방에 메시지를 대신 전달할 수 있습니다.

## 주요 도구 목록

### 1. `sessions_list`
현재 게이트웨이에 존재하는 세션들의 목록을 가져옵니다.
- **필터링**: 메인 채팅, 그룹 채팅, 크론 잡, 노드 세션 등으로 구분하여 조회할 수 있습니다.
- **정보**: 세션 키, 마지막 업데이트 시간, 채팅 채널(Telegram, Slack 등) 정보를 포함합니다.

### 2. `sessions_history`
특정 세션의 대화 기록(트랜스크립트)을 읽어옵니다.
- **매개변수**: 조회할 `sessionKey` 또는 `sessionId`가 필요합니다.
- **제한**: 도구 실행 결과(`toolResult`)를 제외하고 순수 대화 내용만 가져올 수도 있습니다.

### 3. `sessions_send`
다른 세션으로 메시지를 보냅니다. 일종의 "메시지 전달" 기능입니다.
- **상호작용**: 메시지를 보낸 후 상대방 세션의 에이전트가 답변할 때까지 기다렸다가 결과를 받아올 수 있습니다 (핑퐁 루프).
- **활용**: 서로 다른 채널에 있는 대화방 사이의 정보를 동기화하거나 협업할 때 유용합니다.

### 4. `sessions_spawn`
격리된 새로운 세션에서 **서브 에이전트(Sub-agent)**를 실행합니다.
- **비동기 실행**: 서브 에이전트에게 복잡한 작업을 맡기고, 작업이 완료되면 결과만 받아서 원래 대화방에 보고합니다.
- **권한**: 서브 에이전트는 기본적으로 세션 관리 도구를 사용할 수 없도록 격리되어 안전하게 실행됩니다.

## 보안 및 접근 제어

에이전트가 모든 대화방을 마음대로 들여다보지 못하도록 몇 가지 제어 장치가 있습니다:

- **가시성 정책 (Visibility)**:
  - `self`: 현재 세션만 볼 수 있습니다.
  - `tree` (기본값): 현재 세션과 내가 생성한 서브 에이전시 세션만 볼 수 있습니다.
  - `agent`: 같은 에이전트 ID에 속한 모든 세션을 볼 수 있습니다.
  - `all`: 모든 세션에 접근할 수 있습니다.
- **발신 정책 (Send Policy)**: 특정 채널이나 대화 유형(그룹 채팅 등)으로 메시지를 보내는 것을 허용하거나 차단할 수 있습니다.

상세한 API 명세나 설정 방법은 [영문 원본 문서](/concepts/session-tool)를 참고하십시오.
