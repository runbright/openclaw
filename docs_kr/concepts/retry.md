# 재시도 정책 (Retry Policy)

OpenClaw는 메시지 전송이나 API 호출 중에 오류가 발생하면 자동으로 다시 시도하여 서비스의 안정성을 높입니다.

## 주요 목표

- **요청 단위 재시도**: 대화 흐름 전체가 아닌, 실패한 개별 HTTP 요청 단위로 다시 시도합니다.
- **순차적 처리 유지**: 현재 단계가 성공할 때까지 다음 단계로 넘어가지 않아 메시지 순서가 뒤섞이지 않게 합니다.
- **중복 작업 방지**: 이미 성공한 작업(예: 파일 업로드)은 다시 수행하지 않습니다.

## 채널별 동작 방식

### Discord
- **HTTP 429 (속도 제한)** 오류가 발생했을 때만 재시도합니다.
- Discord가 보내주는 `retry_after` 시간을 최우선으로 따르며, 정보가 없으면 지수 백오프(Exponential Backoff) 방식을 사용합니다.

### Telegram
- **일시적인 오류**(속도 제한, 연결 시간 초과, 서버 일시 불능 등) 시 재시도합니다.
- `retry_after` 시간을 따르거나 지수 백오프를 사용합니다.
- 단순한 마크다운 문법 오류는 재시도하지 않고 대신 일반 텍스트 모드로 변환하여 전송합니다.

## 기본 설정값 (Defaults)

- **재시도 횟수**: 3회
- **최대 대기 시간**: 30초
- **지터 (Jitter)**: 0.1 (10% 내외로 대기 시간을 무작위로 조절하여 동시다발적 재시도 방지)

## 설정 방법

`openclaw.json` 파일에서 채널별로 재시도 정책을 직접 설정할 수 있습니다:

```json5
{
  "channels": {
    "telegram": {
      "retry": {
        "attempts": 5,        // 재시도 횟수를 5회로 증가
        "minDelayMs": 500     // 최소 대기 시간 0.5초
      }
    }
  }
}
```
