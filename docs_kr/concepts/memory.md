# 메모리 (Memory)

OpenClaw의 메모리는 **에이전트 워크스페이스에 저장된 일반 텍스트(Markdown) 파일**입니다. 이 파일들이 에이전트 기억의 근원이며, 모델은 디스크에 기록된 내용만을 "기억"합니다.

## 메모리 파일 구조

기본적으로 다음과 같은 두 계층의 메모리 파일을 사용합니다:

- **날짜별 로그 (`memory/YYYY-MM-DD.md`)**
  - 매일매일의 기록이 추가되는 일기 형식의 파일입니다.
  - 새로운 세션이 시작될 때 '오늘'과 '어제'의 기록을 자동으로 읽어옵니다.
- **장기 메모리 (`MEMORY.md`)**
  - (선택 사항) 오랫동안 유지해야 할 핵심 정보나 결정 사항을 정리합니다.
  - **개인적인 비공개 세션에서만 로드**되며, 그룹 채팅 등에서는 노출되지 않습니다.

## 메모리 도구 (Agent Tools)

에이전트는 다음 두 가지 도구를 사용하여 메모리 파일을 관리합니다:

- **`memory_search`**: 시맨틱 검색을 통해 관련 있는 메모리 조각을 찾아냅니다. (말씨가 조금 달라도 의미가 비슷하면 찾아냅니다.)
- **`memory_get`**: 특정 메모리 파일의 내용을 직접 읽어옵니다.

## 자동 메모리 저장 (Memory Flush)

대화가 길어져서 **압축(Compaction)**이 필요해지기 직전에, OpenClaw는 모델에게 중요한 내용을 메모리에 기록하라는 **조용한 신호**를 보냅니다. 이를 통해 대화 내용이 요약되어 사라지기 전에 중요한 사실들을 디스크에 안전하게 저장할 수 있습니다.

## 검색 성능 향상 (하이브리드 검색)

OpenClaw는 두 가지 검색 방식을 결합하여 최적의 결과를 제공합니다:
- **벡터 검색 (Vector Search)**: "문맥이 비슷함"을 인지하여 의미 기반으로 검색합니다.
- **키워드 검색 (BM25)**: 고유 ID, 코드 기호, 특정 단어 등 정확한 일치 항목을 찾아냅니다.

## 사용 팁

- 에이전트에게 **"이 내용을 메모리에 기록해줘"**라고 직접 요청하세요.
- 결정된 사항, 사용자 취향, 반복되는 지침 등은 `MEMORY.md`에 보관하는 것이 좋습니다.
- 매일의 진행 상황이나 스쳐 지나가는 메모는 날짜별 로그(`memory/` 폴더)에 기록하게 하세요.
