# 세션 프루닝 (Session Pruning)

세션 프루닝은 LLM을 호출하기 직전, 메모리에 있는 대화 맥락에서 **오래된 도구 실행 결과**를 정리하는 기능입니다. 이는 디스크에 저장된 실제 대화 기록(`*.jsonl`)을 수정하지 않고, 모델에 보내는 데이터만 일시적으로 줄여줍니다.

## 주요 목적 (비용 및 성능 최적화)

- **컨텍스트 비대화 방지**: 장시간 대화 시 도구 실행 결과(파일 내용, 명령 결과 등)가 누적되어 모델의 기억 용량을 초과하거나 답변 속도가 느려지는 것을 막습니다.
- **캐시 비용 절감**: Anthropic 등의 모델은 프롬프트 캐시 기능을 제공합니다. 프루닝을 통해 캐시 유효 시간(TTL)이 지난 후에 불필요한 데이터를 쳐내면, 다시 캐시를 생성할 때 드는 쓰기 비용을 줄일 수 있습니다.

## 작동 방식

- **대상**: 오직 `toolResult` (도구 실행 결과) 메시지만 정리합니다. 사용자(User)나 에이전트(Assistant)의 대화 내용은 **절대로** 건드리지 않습니다.
- **보호**: 최근에 발생한 몇 개의 에이전트 답변(기본 3개)과 관련된 도구 결과는 삭제되지 않도록 보호합니다.
- **이미지 보호**: 이미지가 포함된 도구 실행 결과는 정보 유지의 중요성 때문에 정리 대상에서 제외됩니다.

## 정리 방법

1. **소프트 트리밍 (Soft-trim)**: 내용이 너무 긴 경우 앞부분과 뒷부분만 남기고 중간을 `...`으로 생략합니다.
2. **하드 클리어 (Hard-clear)**: 아주 오래된 결과는 아예 내용을 지우고 `[오래된 도구 결과 내용이 삭제됨]`과 같은 플레이스홀더로 대체합니다.

## 설정 예시 (`openclaw.json`)

기본적으로 이 기능은 꺼져 있습니다. 다음과 같이 활성활 수 있습니다:

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl", // 캐시 유지 시간이 지났을 때만 실행
        ttl: "5m", // 5분 동안 대화가 없으면 정리 후 실행
        keepLastAssistants: 3, // 마지막 3개의 에이전트 답변 관련 정보는 유지
      }
    }
  }
}
```

상세한 정리 비율이나 도구별 제외 설정 등은 [영문 원본 문서](/concepts/session-pruning)를 참고하십시오.
