# 문맥 (Context)

"문맥"은 **모델이 답변을 생성하기 위해 OpenClaw가 모델에게 전달하는 모든 정보**를 의미합니다. 이 정보의 양은 모델의 **문맥 창(Context Window, 토큰 한도)**에 의해 제한됩니다.

## 문맥의 구성 요소

1. **시스템 프롬프트 (System Prompt)**: OpenClaw가 자동으로 구축하는 규칙, 도구 목록, 스킬 목록, 현재 시간, 그리고 워크스페이스에서 주입된 설정 파일들입니다.
2. **대화 기록 (Conversation History)**: 현재 세션에서 사용자와 에이전트가 나눈 메시지들입니다.
3. **도구 호출 및 결과 (Tool Calls/Results)**: 파일 읽기 결과, 명령어 실행 출력 등 에이전트가 도구를 사용한 상세 내용입니다.
4. **첨부 파일**: 이미지, 오디오, 텍스트 파일 등의 데이터입니다.

## 문맥 확인 및 관리 명령어

- `/status`: 현재 문맥 창이 얼마나 찼는지 요약해서 보여줍니다.
- `/context list`: 주입된 파일 목록과 각각의 크기(토큰 수 추정치)를 보여줍니다.
- `/context detail`: 시스템 프롬프트, 도구 스키마, 스킬 설명 등 상세한 크기 비중을 보여줍니다.
- `/compact`: 오래된 대화 내용을 요약하여 문맥 창의 여유 공간을 확보합니다.

## 워크스페이스 파일 주입 (Project Context)

OpenClaw는 워크스페이스 내의 주요 파일들을 시스템 프롬프트의 '프로젝트 문맥' 섹션에 자동으로 포함시킵니다:
- `AGENTS.md` (운영 지침)
- `SOUL.md` (에이전트 성격)
- `USER.md` (사용자 프로필)
- `IDENTITY.md` (에이전트 식별 정보)
- 등등

파일 내용이 너무 많으면 (`20,000자` 이상) 뒷부분이 잘려서 주입되므로 에이전트가 전체 내용을 읽게 하려면 핵심 위주로 요약하는 것이 좋습니다.

## 도구(Tools)의 두 가지 비용

에이전트가 사용할 수 있는 도구가 많아지면 문맥 창을 많이 차지하게 됩니다:
1. **도구 설명 텍스트**: 시스템 프롬프트에 포함되는 텍스트입니다.
2. **도구 스키마 (JSON)**: 모델이 도구를 어떻게 호출해야 하는지 정의하는 기술적인 데이터입니다. 텍스트로 보이지 않아도 문맥 창 공간을 소모합니다.
