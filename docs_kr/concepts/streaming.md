# 스트리밍 및 청킹 (Streaming & Chunking)

OpenClaw는 답변을 사용자에게 전달할 때 두 가지 층위의 스트리밍 방식을 제공합니다:

1. **블록 스트리밍 (Block Streaming)**: 에이전트가 답변을 작성하는 동안 완성된 **덩어리(블록)** 단위로 메시지를 즉시 보냅니다.
2. **미리보기 스트리밍 (Preview Streaming)**: 답변이 생성되는 동안 임시 메시지를 계속해서 **수정(업데이트)**하며 보여줍니다. (Telegram, Discord, Slack 지원)

## 1. 블록 스트리밍 (메시지 분할 전송)

에이전트의 답변이 길어질 경우, 한꺼번에 보지 않고 작성되는 중간중간 메시지 버블이 하나씩 전송되는 방식입니다.

- **장점**: 에이전트가 긴 글을 쓰는 동안 기다리지 않고 앞부분부터 읽을 수 있습니다.
- **설정**: `agents.defaults.blockStreamingDefault`를 `"on"`으로 설정하여 활성화합니다.
- **조절**: 너무 짧은 조각들이 자주 오지 않도록 `minChars` 등을 설정해 최소 전송 단위를 조절할 수 있습니다.

## 2. 미리보기 스트리밍 (실시간 업데이트)

메시지 버블 하나 안에서 텍스트가 실시간으로 타이핑되듯이 업데이트되는 방식입니다.

- **지원 채널**: Telegram, Discord, Slack
- **모드 종류**:
  - `partial`: 마지막 텍스트로 미리보기 메시지를 계속 교체합니다.
  - `block`: 텍스트 조각을 계속 덧붙여 나갑니다.
  - `progress`: 답변 생성 중에는 진행률/상태를 보여주고, 완료되면 최종 답변으로 바꿉니다.

## 청킹(Chunking) 알고리즘 (글자 수 제한 처리)

각 메신저 채널(WhatsApp 4000자, Discord 2000자 등)의 글자 수 제한을 넘지 않도록 OpenClaw는 스마트하게 메시지를 쪼갭니다:

- **문단/문장 단위 절단**: 가능한 한 문장이나 문단 끝에서 메시지를 나누려고 시도합니다.
- **코드 블록 보호**: ` ``` ` 로 감싸진 코드 블록 중간에서 메시지가 잘리지 않게 보호하며, 어쩔 수 없이 잘려야 할 경우 코드 블록을 닫고 다음 메시지에서 다시 열어 문법을 유지합니다.

## 인간다운 답변 속도 (`humanDelay`)

여러 개의 메시지 버블로 답변할 때, 너무 기계처럼 동시에 전송되지 않도록 메시지 사이에 무작위적인 지연 시간(약 0.8초~2.5초)을 줄 수 있습니다. (`natural` 모드)
