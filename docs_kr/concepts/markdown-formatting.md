# 마크다운 포맷팅 (Markdown Formatting)

OpenClaw는 에이전트가 생성한 마크다운 텍스트를 각 대화 채널(Slack, Telegram, Signal 등)에 최적화된 형식으로 변환하여 전송합니다. 이를 위해 **중간 표현(Intermediate Representation, IR)** 방식을 사용하여 모든 채널에서 일관된 스타일과 링크를 유지하면서도 안전하게 메시지를 분할(Chunking)합니다.

## 주요 목표

- **일관성**: 한 번의 파싱으로 여러 채널의 렌더러에 대응합니다.
- **안전한 메시지 분할**: 굵게(Bold)나 기울임(Italic) 같은 인라인 스타일이 메시지 분할 지점에서 끊어지지 않도록 보호합니다.
- **채널 최적화**: Slack의 mrkdwn, Telegram의 HTML, Signal의 스타일 범위(Style ranges) 등 각 플랫폼의 고유한 문법으로 자동 변환합니다.

## 처리 파이프라인

1. **마크다운 → IR 파싱**: 텍스트를 분석하여 스타일(굵게/코드 등)과 링크 정보가 포함된 IR 객체를 만듭니다.
2. **IR 기반 청킹(Chunking)**: 각 채널의 글자 수 제한에 맞춰 IR 텍스트를 나눕니다. 이때 인라인 스타일은 분할된 각 조각 안에서 다시 열리고 닫히도록 처리됩니다.
3. **채널별 렌더링**:
   - **Slack**: `*굵게*`, `_기울임_`, `<URL|라벨>` 형식으로 변환.
   - **Telegram**: `<b>`, `<i>`, `<code>` 등 HTML 태그 형식으로 변환.
   - **Signal**: 일반 텍스트와 함께 스타일 범위를 지정하거나 `라벨 (URL)` 형식으로 변환.

## 표(Table) 처리 방식

마크다운 표는 모든 채팅 클라이언트에서 동일하게 보이지 않습니다. OpenClaw는 설정을 통해 표를 다음 방식 중 하나로 변환합니다:

- `code` (기본값): 표를 코드 블록 안에 렌더링합니다. (가장 많이 사용됨)
- `bullets`: 각 행을 글머리 기호 목록으로 변환합니다. (Signal, WhatsApp 권장)
- `off`: 변환 없이 원본 마크다운 텍스트를 그대로 보냅니다.

## 링크 정책

- **Slack**: `[라벨](URL)`을 `<URL|라벨>`로 변환합니다.
- **Telegram**: `[라벨](URL)`을 `<a href="URL">라벨</a>`로 변환합니다.
- **Signal**: `[라벨](URL)`을 `라벨 (URL)`로 변환합니다 (라벨과 URL이 다른 경우).

## 주의사항

- **이스케이프(Escape)**: 채널별 문법(예: Slack의 `<`나 `>`, Telegram의 HTML 태그 등)이 오작동하지 않도록 텍스트를 안전하게 처리합니다.
- **동시성**: 메시지가 분할되어 전송될 때 순서가 뒤섞이지 않도록 순차적으로 처리합니다.

상세한 기술 사양이나 설정값은 [영문 원본 문서](/concepts/markdown-formatting)를 참고하십시오.
