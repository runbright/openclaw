# 에이전트 루프 (Agent Loop)

에이전트 루프는 에이전트가 실제로 실행되는 전체 과정을 의미합니다: **입력 수신 → 문맥(Context) 구성 → 모델 추론 → 도구 실행 → 답변 스트리밍 → 데이터 저장**. 이 과정을 통해 사용자의 메시지가 실제 행동과 최종 답변으로 변환되며, 세션의 상태가 일관되게 유지됩니다.

## 실행 단계 (상세 흐름)

1. **요청 수신**: 게이트웨이가 RPC 요청이나 CLI 명령을 통해 에이전트 호출을 받습니다. 세션 정보가 확인되면 즉시 작업 ID(`runId`)를 반환하고 백그라운드에서 실행을 시작합니다.
2. **에이전트 준비**:
   - 사용할 언어 모델과 생각하기(Thinking) 수준을 결정합니다.
   - 스킬(Skills) 정보를 불러와 환경 변수와 프롬프트에 주입합니다.
   - 워크스페이스를 설정하고 프로젝트 파일들을 문맥으로 읽어옵니다.
3. **핵심 실행 루프**:
   - 세션별 큐를 통해 작업을 순차적으로 정렬합니다.
   - 언어 모델(LLM)에 시스템 프롬프트와 대화 기록을 전달합니다.
   - 모델이 도구(Tools)를 호출하면 이를 실행하고 결과를 모델에게 다시 전달합니다.
   - 답변이 생성되는 동안 실시간으로 텍스트를 스트리밍합니다.
4. **결과 처리 및 저장**:
   - 생성된 답변에서 불필요한 중복(도구 사용 알림 등)을 제거합니다.
   - 최종 답변과 사용된 토큰량(Usage)을 기록합니다.
   - 도구 실행 결과를 세션 기록에 저장합니다.

## 주요 특징

### 큐 및 동시성 관리
에이전트 루프는 동일 세션 내에서 작업이 섞이지 않도록 **세션당 하나의 통로(Queue)**를 사용합니다. 이는 대화 기록의 일관성을 유지하고 도구 실행 시 발생할 수 있는 충돌을 방지합니다.

### 스트리밍 (Streaming)
에이전트의 답변은 생성되는 즉시 한 글자씩 또는 문장 단위로 사용자에게 전달됩니다. 이를 통해 사용자는 에이전트가 답변을 완성할 때까지 기다리지 않고 실시간으로 진행 상황을 확인할 수 있습니다.

### 후킹 및 인터셉트 (Hooks)
개발자는 훅(Hooks) 시스템을 통해 에이전트 루프의 특정 시점에 개입할 수 있습니다:
- **`agent:bootstrap`**: 시스템 프롬프트가 완성되기 전에 문맥 파일을 추가/삭제합니다.
- **`before_model_resolve`**: 모델이 결정되기 전에 사용할 모델(GPT, Claude 등)을 강제로 변경합니다.
- **`tool_result_persist`**: 도구 실행 결과가 기록되기 전에 데이터를 변환합니다.

## 실행 중단 상황

- **타임아웃(Timeout)**: 설정된 시간(기본 600초) 내에 작업이 끝나지 않으면 강제 종료됩니다.
- **중단 신호(Abort)**: 사용자가 `/stop` 명령을 내리거나 클라이언트 연결이 끊어지면 실행이 취소됩니다.
