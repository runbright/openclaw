# 문맥 창 및 압축 (Compaction)

모든 인공지능 모델은 한 번에 볼 수 있는 정보의 양인 **문맥 창(Context Window, 토큰 한도)**을 가지고 있습니다. 대화가 길어지면 메시지와 도구 실행 결과가 쌓여 이 한도를 넘게 되는데, 이때 OpenClaw는 오래된 기록을 **압축(Compact)**하여 한도 내로 유지합니다.

## 압축이란 무엇인가요?

압축은 **이전의 긴 대화 내용을 요약본 하나로 줄이고**, 최근 메시지들은 그대로 유지하는 과정입니다. 요약된 내용은 세션 기록에 저장되며, 이후 모델은 다음 정보를 바탕으로 답변을 생성합니다:

1. **압축 요약본**: 이전 대화의 핵심 내용
2. **최근 메시지들**: 압축된 시점 이후의 대화

이 과정은 세션의 기록 파일(JSONL)에 영구적으로 저장됩니다.

## 주요 기능

### 1. 자동 압축 (Auto-compaction)
대화 내용이 모델의 한도에 가까워지면 OpenClaw가 자동으로 압축을 실행합니다. 설정 파일에서 `agents.defaults.compaction` 항목을 통해 동작 방식을 조절할 수 있습니다.

### 2. 수동 압축 (`/compact`)
사용자가 직접 `/compact` 명령어를 입력하여 대화를 압축할 수 있습니다. 이때 요약 시 중점적으로 다룰 내용을 함께 전달할 수 있습니다:
> `/compact 지금까지 결정된 사항들과 남은 질문들 위주로 요약해줘`

## 압축 vs 가다듬기 (Pruning)

- **압축 (Compaction)**: 대화 전체를 요약하여 **영구적으로 기록**합니다.
- **세션 가다듬기 (Session Pruning)**: 매 요청마다 **메모리 상에서만** 오래된 **도구 실행 결과**를 잘라냅니다. (자세한 내용은 [세션 가다듬기](/concepts/session-pruning) 참고)

## 팁

- 대화 맥락이 너무 복잡해졌거나 에이전트의 반응이 느려진 것 같을 때 `/compact`를 사용해 보세요.
- 완전히 새로운 대화를 시작하고 싶다면 `/new` 또는 `/reset` 명령어를 사용하여 새 세션을 시작하는 것이 좋습니다.
