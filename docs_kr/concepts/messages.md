# 메시지 (Messages)

OpenClaw가 수신된 메시지를 어떻게 처리하고, 세션을 관리하며, 답변을 전송하는지에 대한 전반적인 프로세스를 설명합니다.

## 메시지 흐름 (High Level)

1. **메시지 수신**: 각 채널(WhatsApp, Telegram 등)로부터 메시지가 도착합니다.
2. **라우팅 및 바인딩**: 메시지 발신자와 채널 정보를 바탕으로 어떤 에이전트 세션으로 보낼지 결정합니다.
3. **대기열(Queue) 처리**: 이미 에이전트가 다른 작업을 수행 중이라면, 설정된 대기열 정책에 따라 대기하거나 현재 작업을 중단/수정합니다.
4. **에이전트 실행**: 모델이 메시지를 분석하고, 필요시 도구를 실행하며 답변을 생성합니다.
5. **답변 전송**: 생성된 답변을 채널별 글자 수 제한 등에 맞춰 쪼개거나 스트리밍 방식으로 사용자에게 전달합니다.

## 주요 기능 및 설정

### 1. 중복 제거 및 디바운싱 (Deduplication & Debouncing)
- **중복 제거**: 네트워크 불안정으로 동일한 메시지가 여러 번 도착하는 것을 방지합니다.
- **디바운싱**: 짧은 시간 내에 여러 개의 텍스트 메시지를 보내면 하나의 덩어리로 묶어서 에이전트에게 전달합니다. (기본 2초 대기)

### 2. 세션 관리 (Sessions)
- 개인 채팅은 기본적으로 에이전트의 '메인 세션'으로 통합됩니다.
- 그룹 채팅이나 채널은 각각 독립된 세션 키를 가지며 대화 맥락이 분리됩니다.
- 모든 대화 기록은 게이트웨이가 실행되는 컴퓨터의 디스크에 안전하게 저장됩니다.

### 3. 대기열 및 후속 작업 (Queueing)
에이전트가 답변을 작성 중일 때 추가 메시지가 오면 어떻게 할지 선택할 수 있습니다:
- `interrupt`: 현재 작업을 즉시 멈추고 새 메시지를 처리합니다.
- `steer`: 답변 방향을 새 메시지에 맞춰 수정합니다.
- `followup`: 현재 답변이 끝난 후 바로 이어서 다음 메시지를 처리합니다.

### 4. 생각하기 과정 노출 (`/reasoning`)
일부 모델(예: Claude, GPT-o1)의 '생각하기' 단계를 사용자에게 보여줄지 결정할 수 있습니다.
- `/reasoning on`: 생각 과정을 함께 보여줍니다.
- `/reasoning off`: 최종 결과만 보여줍니다.

## 메시지 본문 구성 (Prompt Body)

에이전트에게 전달되는 메시지에는 단순 텍스트 외에 맥락 정보가 포함되기도 합니다:
- `[이전 답변 이후의 메시지들 - 참고용]`
- `[현재 메시지 - 이에 대해 응답하세요]`

이러한 구조를 통해 에이전트는 사용자가 여러 번 메시지를 보내거나 그룹 채팅에서 다른 사람들이 말한 내용을 정확히 파악할 수 있습니다.
