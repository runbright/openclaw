# 에이전트 워크스페이스 (Agent Workspace)

워크스페이스는 에이전트의 '집'이자 작업 공간입니다. 에이전트가 파일을 읽고 쓰고 수정하는 모든 활동이 이루어지는 전용 디렉토리입니다. 이곳의 파일들은 에이전트의 기억 장치와 같으므로 비공개로 안전하게 관리해야 합니다.

*이 공간은 설정 파일이나 인증 정보가 저장되는 `~/.openclaw/` 폴더와는 별도로 관리됩니다.*

## 기본 위치

- **기본 경로**: `~/.openclaw/workspace`
- **설정 변경**: `~/.openclaw/openclaw.json` 파일에서 경로를 수정할 수 있습니다:

```json5
{
  "agent": {
    "workspace": "~/.openclaw/workspace"
  }
}
```

## 워크스페이스 파일 구성 (주요 파일 설명)

OpenClaw 워크스페이스 안에는 다음과 같은 표준 파일들이 포함됩니다:

- **`AGENTS.md`**: 에이전트가 지켜야 할 운영 규칙과 메모리 사용법이 적힌 설명서입니다.
- **`SOUL.md`**: 에이전트의 성격, 말투, 대화의 한계 등을 정의합니다.
- **`USER.md`**: 사용자가 누구인지, 대상을 어떻게 불러야 하는지 등에 대한 정보입니다.
- **`IDENTITY.md`**: 에이전트의 이름, 상징하는 이모지, 전반적인 분위기를 설정합니다.
- **`TOOLS.md`**: 등록된 도구(Tools)의 사용법이나 주의사항을 적어두는 곳입니다.
- **`MEMORY.md`**: (선택 사항) 에이전트가 오랫동안 기억해야 할 핵심 정보들을 정리해두는 공간입니다.
- **`memory/` 폴더**: 날짜별(예: `2026-01-20.md`)로 그날의 대화 포인트나 중요 사건이 기록되는 일기장 같은 공간입니다.

## 백업 전략 (Git 활용 권장)

워크스페이스는 에이전트의 소중한 기억이 담긴 곳이므로, **비공개(Private) Git 저장소**를 만들어 백업하는 것을 강력히 추천합니다.

1. **저장소 초기화**:
   ```bash
   cd ~/.openclaw/workspace
   git init
   git add .
   git commit -m "첫 번째 워크스페이스 백업"
   ```
2. **원격 저장소 연결**: GitHub나 GitLab에서 **비공개(Private)** 저장소를 만든 후 연결하여 푸시(Push)하세요.

## 보안 주의사항

- **비밀번호 저장 금지**: 워크스페이스 파일(Markdown) 내에 API 키, 비밀번호 등 민감한 정보를 직접 적지 마세요.
- **비공개 유지**: Git 저장소는 반드시 **비공개 모드**로 설정하여 타인이 에이전트의 기억에 접근하지 못하게 해야 합니다.
