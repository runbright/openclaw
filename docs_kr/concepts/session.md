# 세션 관리 (Session Management)

OpenClaw는 대화의 맥락을 유지하기 위해 "세션(Session)"이라는 개념을 사용합니다. 각 세션은 고유한 식별자(Session Key)를 가지며, 대화 기록은 독립적인 파일로 관리됩니다.

## 대화 격리 및 보안 (Secure DM 모드)

> **보안 경고**: 에이전트가 **여러 명의 사용자**로부터 메시지를 받는 경우, 반드시 세션 격리 설정을 확인해야 합니다. 기본 설정으로는 모든 사용자가 하나의 대화 맥락을 공유하게 되어 개인 정보가 노출될 위험이 있습니다.

**해결 방법**: `dmScope` 설정을 변경하여 사용자별로 대화방을 분리하세요.

```json5
// ~/.openclaw/openclaw.json
{
  "session": {
    // 사용자별(발신자별)로 대화방을 완전히 분리합니다.
    "dmScope": "per-channel-peer"
  }
}
```

## 세션 저장소 위치

모든 세션 정보와 대화 기록은 게이트웨이가 실행되는 컴퓨터에 저장됩니다:
- **전체 세션 정보**: `~/.openclaw/agents/<에이전트ID>/sessions/sessions.json`
- **개별 대화 기록**: `~/.openclaw/agents/<에이전트ID>/sessions/<세션ID>.jsonl`

## 세션 초기화 및 만료 규칙 (Lifecycle)

- **언제 세션이 새로 시작되나요?**: 사용자가 `/new` 또는 `/reset` 명령어를 입력했을 때, 혹은 설정된 시간이 지나 세션이 만료되었을 때 새로 시작됩니다.
- **매일 초기화**: 기본적으로 **매일 새벽 4시(게이트웨이 실행 컴퓨터 시간)**에 대화 기록이 만료되어, 다음날 첫 메시지를 보낼 때 새로운 세션으로 인식됩니다.
- **유휴 시간 만료**: 일정 시간(`idleMinutes`) 동안 대화가 없으면 세션을 만료하도록 설정할 수도 있습니다.

## 유지 관리 (Maintenance)

대화 기록이 무한정 쌓여 디스크 용량을 차지하는 것을 방지하기 위해 자동 정리 기능을 제공합니다:
- **`pruneAfter`**: 일정 기간(예: 30일)이 지난 세션을 정리합니다.
- **`maxEntries`**: 최대 보유할 세션 개수를 제한합니다. (기본 500개)
- **`maxDiskBytes`**: 세션 저장소가 차지할 수 있는 최대 디스크 용량을 설정합니다.

## 세션 확인 명령어

- `/status`: 현재 세션의 토큰 사용량과 설정 상태를 채팅창에서 즉시 확인합니다.
- `/stop`: 현재 진행 중인 에이전트 작업을 즉시 중단합니다.
- `/compact`: 긴 대화 내용을 요약하여 문맥 창을 비웁니다.
