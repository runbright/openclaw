# 멀티 에이전트 라우팅 (Multi-Agent Routing)

OpenClaw는 하나의 게이트웨이 서버에서 여러 명의 독립된 '에이전트'를 동시에 운영할 수 있습니다. 각 에이전트는 자신만의 워크스페이스, 대화 기록, 그리고 API 키를 가집니다.

## "하나의 에이전트"란 무엇인가요?

에이전트는 독립된 '뇌'를 가진 개체입니다:
- **워크스페이스**: 에이전트의 성격(`SOUL.md`), 규칙(`AGENTS.md`), 그리고 로컬 파일들.
- **인증 프로필**: 에이전트별로 사용하는 별도의 API 키와 계정 정보.
- **세션 저장소**: 개별적인 대화 기록 및 기록 파일.

## 멀티 에이전트의 이점

1. **완벽한 페르소나 분리**: 업무용(Work) 에이전트와 개인용(Home) 에이전트를 같은 서버에서 분리하여 운영할 수 있습니다.
2. **다중 계정 지원**: 하나의 서버에서 두 개의 서로 다른 WhatsApp 번호나 텔레그램 봇을 각각 다른 에이전트에 연결할 수 있습니다.
3. **데이터 격리**: 여러 사람이 하나의 게이트웨이를 공유하더라도 각자의 에이전트와 나눈 대화 정보는 서로 섞이지 않습니다.

## 라우팅 설정 방법 (Bindings)

수신된 메시지를 어떤 에이전트에게 보낼지는 `bindings` 설정을 통해 결정합니다.

```json5
{
  "agents": {
    "list": [
      { "id": "coding", "workspace": "~/.openclaw/workspace-coding" },
      { "id": "social", "workspace": "~/.openclaw/workspace-social" }
    ]
  },
  "bindings": [
    // 텔레그램으로 오는 메시지는 'social' 에이전트에게 전송
    { "agentId": "social", "match": { "channel": "telegram" } },
    // 왓츠앱으로 오는 메시지는 'coding' 에이전트에게 전송
    { "agentId": "coding", "match": { "channel": "whatsapp" } }
  ]
}
```

### 라우팅 우선순위
메시지가 도착하면 OpenClaw는 다음 기준에 따라 가장 적합한 에이전트를 찾습니다:
1. **특정 대화 상대 ID**와 일치하는가? (가장 높은 우선순위)
2. **채널 계정 ID**가 일치하는가? (예: 특정 왓츠앱 번호)
3. **채널 종류**가 일치하는가? (예: 모든 텔레그램 메시지)
4. 일치하는 항목이 없다면 **기본(Default) 에이전트**에게 보냅니다.

## 유용한 관리 명령어

- **에이전트 추가**: `openclaw agents add [이름]`
- **설정 확인**: `openclaw agents list --bindings` (각 계정과 에이전트가 어떻게 연결되어 있는지 한눈에 보여줍니다.)
