# 타이핑 표시 (Typing Indicators)

에이전트가 답변을 생성하는 동안 채팅 채널에 "타이핑 중..." 표시를 보냅니다. `agents.defaults.typingMode` 설정을 통해 **언제** 타이핑 표시를 시작할지 제어할 수 있고, `typingIntervalSeconds`를 통해 표시가 사라지지 않도록 얼마나 자주 **갱신**할지 설정할 수 있습니다.

## 기본 동작 (별도 설정이 없을 때)

- **개인 채팅**: 모델 실행 루프가 시작되면 즉시 타이핑 표시를 보냅니다.
- **그룹 채팅 (멘션 포함)**: 에이전트가 호출되면 즉시 타이핑 표시를 시작합니다.
- **그룹 채팅 (멘션 미포함)**: 실제로 답변 텍스트가 스트리밍되기 시작할 때만 타이핑 표시를 보여줍니다.
- **하트비트(Heartbeat) 실행**: 타이핑 표시가 나타나지 않습니다.

## 타이핑 모드 (Typing Modes)

다음 중 하나를 선택하여 `agents.defaults.typingMode`에 설정할 수 있습니다:

- **`never`**: 어떤 상황에서도 타이핑 표시를 하지 않습니다.
- **`instant`**: 모델이 생각을 시작하자마자 즉시 타이핑 표시를 보냅니다. (가장 빠름)
- **`thinking`**: 모델의 **추론 과정(Reasoning delta)**이 스트리밍되기 시작할 때 타이핑 표시를 보냅니다.
- **`message`**: 실제로 사용자에게 전달될 **답변 텍스트**가 나오기 시작할 때 타이핑 표시를 보여줍니다. (무응답 토큰 `NO_REPLY`는 무시함)

## 설정 방법 (`openclaw.json`)

```json5
{
  agent: {
    typingMode: "thinking", // 추론 시작 시 표시
    typingIntervalSeconds: 6, // 6초마다 타이핑 상태 갱신
  },
}
```

## 주의 사항

- **`message` 모드**: 에이전트가 답변을 하지 않기로 결정한 경우(Silent reply)에는 타이핑 표시가 나타나지 않아 더 자연스럽습니다.
- **`thinking` 모드**: 에이전트 설정에서 추론 스트리밍(`reasoningLevel: "stream"`)이 활성화되어 있어야 동작합니다.
- **주기 설정**: `typingIntervalSeconds`는 타이핑을 시작하는 시간이 아니라, 메신저 플랫폼에서 타이핑 표시가 꺼지지 않도록 다시 보내는 주기를 의미합니다.

상세 기술 사양은 [영문 원본 문서](/concepts/typing-indicators)를 참고하십시오.
