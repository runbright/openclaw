# 접속 상태 (Presence)

OpenClaw의 "접속 상태(Presence)"는 시스템의 현재 상황을 가볍게 파악하기 위한 기능입니다. 주로 다음 두 가지 정보를 보여줍니다:

- **게이트웨이(Gateway)** 자체의 상태
- **게이트웨이에 연결된 클라이언트**들 (macOS 앱, 웹 대화창, CLI 등)

이 정보는 macOS 앱의 **인스턴스(Instances)** 탭에서 시각적으로 확인할 수 있어, 현재 어떤 기기들이 연결되어 있는지 한눈에 파악하는 데 도움을 줍니다.

## 포함되는 정보들

접속 상태 항목에는 다음과 같은 정보들이 포함됩니다:
- **호스트 이름**: 연결된 컴퓨터나 기기 이름
- **IP 주소**: 현재 접속 중인 네트워크 주소
- **버전**: 클라이언트 앱의 버전 정보
- **모드**: 사용 중인 환경 (UI, 웹, CLI, 노드 등)
- **최종 입력 시간**: 사용자가 마지막으로 조작한 지 얼마나 지났는지 정보
- **업데이트 시간**: 정보가 마지막으로 갱신된 시각

## 정보 수집 및 관리 방식

1. **자동 생성**: 게이트웨이가 시작되면 자동으로 "자기 자신(Gateway)" 정보를 먼저 생성합니다.
2. **연결 시 등록**: 새로운 클라이언트가 WebSocket을 통해 접속하면 즉시 목록에 추가됩니다.
3. **주기적 갱신**: 클라이언트는 자신의 상태(이름, IP, 유휴 상태 등)를 주기적으로 게이트웨이에 보고합니다.
4. **유지 기간(TTL)**: 접속 상태 정보는 휘발성 데이터입니다. 5분 이상 업데이트가 없는 정보는 자동으로 목록에서 삭제됩니다.

## 중복 표시 문제 해결

만약 인스턴스 목록에 동일한 기기가 여러 번 나타난다면, 클라이언트가 고유 식별자(`instanceId`)를 매번 다르게 생성하고 있을 가능성이 큽니다. 안정적인 연결을 위해서는 클라이언트가 재접속 시에도 동일한 식별자를 유지해야 합니다.
