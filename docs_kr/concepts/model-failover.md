# 모델 오류 복구 (Model Failover)

OpenClaw는 모델 실행 중 오류가 발생하면 두 단계에 걸쳐 자동으로 복구를 시도합니다:

1. **인증 프로필 전환**: 현재 사용 중인 모델 제공사(Provider) 내에서 다른 계정이나 API 키로 전환합니다.
2. **모델 대체 (Fallback)**: 해당 제공사의 모든 계정이 실패하면, 설정된 '대체 모델' 목록 중 다음 모델로 전환합니다.

## 1단계: 인증 프로필 전환 (Auth Profile Rotation)

OpenClaw는 같은 서비스(예: Google, OpenAI)에 대해 여러 개의 API 키나 OAuth 계정을 등록할 수 있습니다. 한 계정에서 속도 제한(Rate Limit)이 걸리면 다음 순서로 계정을 전환합니다:

- **OAuth 계정 우선**: API 키보다 OAuth 연동 계정을 먼저 사용합니다.
- **오래된 계정 우선**: 사용한 지 가장 오래된 계정부터 순환하며 사용합니다.
- **세션 고정 (Stickiness)**: 효율성을 위해 한 대화 세션 중에는 가능한 한 동일한 계정을 계속 사용합니다. (대화가 새로 시작되거나 보관 처리가 되면 변경될 수 있습니다.)

## 2단계: 대기 시간 및 비활성화 (Cooldowns)

오류의 성격에 따라 해당 계정을 일시적으로 또는 장기적으로 사용 중단합니다.

- **일시 대기 (Cooldown)**: 속도 제한이나 일시적인 네트워크 오류 발생 시.
  - 대기 시간은 1분 → 5분 → 25분 → 최대 1시간으로 점진적으로 늘어납니다.
- **장기 비활성화 (Disabled)**: 잔액 부족(Billing) 등 즉시 해결되지 않는 문제 발생 시.
  - 기본 5시간 동안 해당 계정을 사용하지 않으며, 반복 실패 시 최대 24시간까지 중단됩니다.

## 3단계: 모델 대체 (Model Fallback)

현재 모델의 모든 계정에서 실패가 발생하면 설정 파일(`openclaw.json`)의 `fallbacks` 목록에 정의된 다음 모델을 사용합니다.

```json5
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "google/gemini-1.5-pro",
        "fallbacks": ["google/gemini-1.5-flash", "openai/gpt-4o-mini"]
      }
    }
  }
}
```

예를 들어 위 설정에서는 `Gemini Pro`가 실패하면 `Flash` 모델을 시도하고, 이마저도 실패하면 `GPT-4o mini`를 사용하여 대화를 이어갑니다.
