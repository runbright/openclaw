# 에이전트 런타임 (Agent Runtime) 🤖

OpenClaw는 **pi-mono** 아키텍처를 기반으로 한 단일 내장 에이전트 엔진을 통해 작동합니다.

## 워크스페이스 (필수)

에이전트는 지정된 단일 디렉토리를 작업 공간(`cwd`)으로 삼아 도구를 실행하고 파일을 관리합니다. 처음 시작한다면 `openclaw setup` 명령어를 통해 필요한 설정 파일과 워크스페이스 구조를 자동으로 만드는 것이 좋습니다.

상세 워크스페이스 구조 및 백업 방법은 [에이전트 워크스페이스](/concepts/agent-workspace)를 참고하세요.

## 자동 주입되는 초기 파일들 (Bootstrap Files)

워크스페이스 내의 다음 파일들은 새로운 세션이 시작될 때 에이전트의 기억(Context)에 자동으로 주입됩니다:

- **`AGENTS.md`**: 행동 지침 및 메모리 사용 규칙
- **`SOUL.md`**: 페르소나, 어조, 대화의 경계 설정
- **`USER.md`**: 사용자 프로필 및 선호하는 호칭
- **`IDENTITY.md`**: 에이전트의 이름과 상징하는 이모지
- **`TOOLS.md`**: 사용 환경에 맞춘 도구 사용 가이드

파일이 너무 크면 자동으로 잘려서 주입되므로, 핵심 내용은 요약하여 작성하는 것이 효율적입니다.

## 스킬 (Skills)

OpenClaw는 다음 세 경로에서 스킬을 불러옵니다:
1. **Bundled**: 설치 시 기본으로 제공되는 스킬들
2. **Managed**: `~/.openclaw/skills`에 사용자가 직접 추가한 스킬들
3. **Workspace**: 워크스페이스 내부의 `skills` 폴더 (가장 높은 우선순위)

## 세션 관리 (Sessions)

대화 기록은 JSONL 형식으로 다음 경로에 안전하게 저장됩니다:
`~/.openclaw/agents/<에이전트ID>/sessions/`

## 답변 생성 방식 (Streaming & Steering)

- **스티어링 (Steering)**: 에이전트가 답변을 생성하는 도중에도 새로운 사용자 메시지가 도착하면, 에이전트가 이를 인지하고 답변 방향을 실시간으로 수정할 수 있습니다.
- **스트리밍 (Streaming)**: 답변이 완성될 때까지 기다리지 않고 실시간으로 한 글자씩 사용자에게 보여줍니다. (자세한 내용은 [스트리밍 가이드](/concepts/streaming) 참고)

## 최소 필수 설정

시스템이 정상 작동하기 위해 최소한 다음 두 가지는 설정해야 합니다:
1. `agents.defaults.workspace`: 에이전트 작업 공간 경로
2. `channels.whatsapp.allowFrom`: 에이전트와 대화할 수 있는 전화번호
