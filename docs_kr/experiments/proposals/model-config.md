# 모델 설정 (탐구)

이 문서는 향후 모델 구성에 대한 **아이디어**를 기록합니다. 배포용 사양이 아닙니다.
현재 동작에 대해서는 다음을 참조하세요:

- [모델](/concepts/models)
- [모델 장애 조치](/concepts/model-failover)
- [OAuth + 프로필](/concepts/oauth)

## 동기

운영자가 원하는 것:

- 프로바이더별 다중 인증 프로필 (개인 vs 업무).
- 예측 가능한 폴백을 가진 간단한 `/model` 선택.
- 텍스트 모델과 이미지 지원 모델 간의 명확한 분리.

## 가능한 방향 (상위 수준)

- 모델 선택을 단순하게 유지: 선택적 별칭이 있는 `provider/model`.
- 프로바이더가 다중 인증 프로필을 가질 수 있게 하되 명시적 순서로.
- 모든 세션이 일관되게 장애 조치되도록 글로벌 폴백 목록 사용.
- 명시적으로 구성된 경우에만 이미지 라우팅 재정의.

## 미해결 질문

- 프로필 순환은 프로바이더별이어야 하는가 모델별이어야 하는가?
- UI가 세션에 대한 프로필 선택을 어떻게 표시해야 하는가?
- 레거시 설정 키에서의 가장 안전한 마이그레이션 경로는 무엇인가?
