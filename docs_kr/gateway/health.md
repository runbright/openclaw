# 상태 점검 (Health Checks)

OpenClaw의 연결 상태를 빠르고 정확하게 진단하기 위한 가이드입니다.

## 빠른 점검 방법

- **`openclaw status`**: 로컬 요약을 확인합니다. 게이트웨이 접속 가능 여부, 연결된 채널의 인증 정보 최신화 상태, 최근 세션 활동 등을 보여줍니다.
- **`openclaw status --deep`**: 실행 중인 게이트웨이에 직접 질의하여 채널별 상세 연결 상태를 점검합니다.
- **`openclaw health --json`**: 게이트웨이로부터 전체 건강 상태 스냅샷을 가져옵니다.
- **명령어 입력**: 채팅창에 `/status` 메시지를 보내면 에이전트를 호출하지 않고도 현재 봇의 상태 답장을 받을 수 있습니다.

## 심층 진단 테마

- **인증 파일 확인**: `~/.openclaw/credentials/` 폴더 내의 각 채널 인증 정보(`creds.json` 등)의 수정 시간이 최근인지 확인하세요.
- **세션 저장소**: 세션 기록이 누적되고 있는지 확인하고, 누락된 것이 있다면 `status` 명령어로 최근 수신인 목록을 대조해보십시오.
- **재연결 흐름**: 로그에 409~515 에러 코드나 `loggedOut`이 나타나면 `openclaw channels logout` 후 `openclaw channels login`을 통해 다시 연동해야 합니다.

## 실패 시 대처법

- **로그아웃 상태**: `openclaw channels logout` 후 다시 로그인하십시오.
- **게이트웨이 접속 불가**: `openclaw gateway --port 18789` 명령어로 게이트웨이를 수동으로 실행하십시오.
- **메시지 수신 불가**: 연결된 휴대전화가 온라인인지, 보낸 사람이 허용 목록(`allowFrom`)에 포함되어 있는지, 그룹 채팅의 경우 멘션 규칙(`requireMention`)에 맞게 보냈는지 확인하십시오.

상세한 CLI 옵션이나 에러 코드 설명은 [영문 원본 문서](/gateway/health)를 참고하십시오.
