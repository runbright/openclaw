# 탐색 및 전송 방식 (Discovery & Transports)

OpenClaw는 서로 다른 두 가지 연결 문제를 해결하도록 설계되었습니다:
1. **관리자 원격 제어**: 다른 곳에서 실행 중인 게이트웨이를 macOS 메뉴 바 앱으로 제어하는 경우.
2. **노드 페어링**: iOS/Android 장치가 게이트웨이를 찾아 안전하게 연결(페어링)하는 경우.

모든 네트워크 탐색과 광고 기능은 **노드 게이트웨이**(`openclaw gateway`)가 담당하며, 클라이언트(macOS 앱, iOS 앱 등)는 이를 소비하여 연결합니다.

## 주요 연결 방식

### 1. 직접 WebSocket 연결 (Direct WS)
동일한 네트워크(LAN)나 Tailscale 환경에서 가장 추천되는 방식입니다.
- **장점**: Bonjour를 통한 자동 탐색, 게이트웨이가 직접 관리하는 페어링 토큰 및 접근 제어(ACL), 별도의 SSH 계정 불필요.
- **환경**: LAN 내에서는 Bonjour를 사용하고, 네트워크를 넘나들 때는 Tailscale의 고정 IP나 MagicDNS를 사용합니다.

### 2. SSH 터널링 (SSH Transport)
직접 연결이 불가능한 환경에서 사용하는 범용적인 대체 방식입니다.
- **장점**: SSH 접근만 가능하면 어디서든 작동하며, Bonjour가 작동하지 않는 네트워크 환경에서도 신뢰할 수 있습니다.
- **보안**: 별도의 포트를 열지 않고 기존 SSH 포트만 사용하여 게이트웨이 포트(`18789`)를 포워딩합니다.

## 탐색 경로 (Discovery)

### Bonjour / mDNS (LAN 전용)
같은 로컬 네트워크에 있는 게이트웨이를 자동으로 찾아주는 편의 기능입니다.
- **게이트웨이**: Bonjour를 통해 자신의 위치를 알립니다(Beacon).
- **클라이언트**: 목록에서 탐색된 게이트웨이를 선택하여 저장합니다.

> **상세 정보**: 게이트웨이는 자신의 역할, 호스트 이름, 포트 번호, TLS 활성화 여부 등을 비인증 메타데이터(TXT 레코드)로 배포합니다. 자세한 디버깅 방법은 [Bonjour 문서](/gateway/bonjour)를 참고하십시오.

### Tailscale (네트워크 경계 돌파)
네트워크가 서로 다른 경우(예: 집과 사무실) Bonjour 대신 Tailscale의 MagicDNS 이름을 사용하여 직접 연결하는 것을 권장합니다. 게이트웨이가 Tailscale 환경임을 감지하면 관련 힌트를 클라이언트에 제공합니다.

## 연결 우선순위 (Client Policy)

클라이언트 앱은 보통 다음 순서로 연결을 시도합니다:
1. **저장된 직접 엔드포인트**: 이전에 페어링된 정보가 있고 연결 가능하다면 사용합니다.
2. **Bonjour 탐색**: 로컬 네트워크에서 게이트웨이가 발견되면 사용자에게 "이 게이트웨이 사용"을 제안합니다.
3. **Tailscale/수동 주소**: 설정된 Tailscale 주소나 수동 IP로 연결을 시도합니다.
4. **SSH 대체**: 모든 직접 연결이 실패하면 최후의 수단으로 SSH 터널링을 시도합니다.

상세한 페어링 절차는 [게이트웨이 페어링 문서](/gateway/pairing)를, 프로토콜 명세는 [게이트웨이 프로토콜 문서](/gateway/protocol)를 참고하십시오.
