# 게이트웨이 잠금 (Gateway Lock)

본 문서는 한 시스템 내에서 여러 게이트웨이가 동시에 실행되어 충돌하는 것을 방지하는 메커니즘을 설명합니다.

## 잠금이 필요한 이유

- **중복 방지**: 한 호스트에서 동일한 포트로 여러 게이트웨이가 작동하여 데이터가 꼬이는 것을 방지합니다.
- **안정성**: 프로세스가 갑자기 종료되거나 강제 종료(`SIGKILL`)되어도 찌꺼기(stale) 잠금 파일이 남지 않도록 합니다.
- **빠른 피드백**: 포트가 이미 사용 중이라면 즉시 명확한 에러 메시지를 보여줍니다.

## 작동 방식

- **포트 선점**: 게이트웨이는 시작 직후 즉시 WebSocket 리스너(기본값 `18789` 포트)를 독점적으로 바인딩합니다.
- **충돌 감지**: 이미 포트가 사용 중이라면(`EADDRINUSE`), "이미 다른 게이트웨이 인스턴스가 해당 포트에서 실행 중입니다"라는 에러와 함께 즉시 종료됩니다.
- **자동 해제**: 프로세스가 종료되면 OS 레벨에서 해당 포트를 즉시 개방하므로 별도의 잠금 파일 삭제 과정이 필요 없습니다.

## 운영 참고 사항

- **포트 점유**: 게이트웨이가 아닌 다른 프로그램이 해당 포트를 쓰고 있어도 동일한 잠금 에러가 발생합니다. 이 경우 해당 포트를 비우거나 `openclaw gateway --port <다른포트>` 명령어로 다른 포트를 지정해야 합니다.
- **macOS 앱**: 메뉴 바 앱은 자체적으로 PID를 감시하여 게이트웨이를 중복 실행하지 않도록 한 번 더 보호합니다.

상세한 에러 메시지 형식 등은 [영문 원본 문서](/gateway/gateway-lock)를 참고하십시오.
