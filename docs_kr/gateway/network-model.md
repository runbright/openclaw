# 네트워크 모델 (Network Model)

OpenClaw의 모든 통신은 게이트웨이(`openclaw gateway`)를 중심으로 이루어집니다. 게이트웨이는 채널 연결(WhatsApp 등)과 WebSocket 제어판을 담당하는 핵심 프로세스입니다.

## 핵심 규칙

1. **단일 게이트웨이 지향**: 한 호스트당 하나의 게이트웨이 실행을 권장합니다. 이는 WhatsApp 세션 중복을 방지하기 위함입니다. (격리가 필요한 경우 [다중 게이트웨이](/gateway/multiple-gateways) 운영이 가능합니다.)
2. **로컬 우선 (Loopback first)**: 게이트웨이 WebSocket은 기본적으로 `127.0.0.1:18789` 주소로 작동합니다. 보안을 위해 루프백 접속이라도 인증 토큰 사용을 권장합니다.
3. **외부 접속 시 인증 필수**: Tailscale이나 LAN을 통해 외부에서 접속하도록 바인딩(`--bind tailnet` 등)할 경우, **반드시 인증 토큰(`--token`)이 설정되어야 합니다.**
4. **노드 연결**: iOS/Android 앱 등 클라이언트 노드는 LAN, Tailscale 또는 SSH 터널을 통해 게이트웨이 WebSocket에 접속합니다.
5. **통합 서비스**: 캔버스(Canvas) 호스트 등 부가 서비스는 게이트웨이와 **동일한 포트**(기본 18789)의 특정 웹 경로(`/__openclaw__/canvas/` 등)를 통해 제공됩니다.
6. **원격 접근**: 직접 포트를 여는 대신 Tailscale VPN이나 SSH 터널링을 사용하는 것이 가장 안전하고 권장되는 방법입니다.

상세한 탐색 정책은 [탐색 가이드](/gateway/discovery)를, 원격 설정은 [원격 접속 가이드](/gateway/remote)를 참고하십시오.
