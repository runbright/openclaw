# 백그라운드 실행 및 프로세스 도구

OpenClaw는 `exec` 도구를 통해 쉘 명령어를 실행하며, 시간이 오래 걸리는 작업은 백그라운드 프로세스로 유지할 수 있습니다. `process` 도구는 이렇게 실행 중인 백그라운드 세션들을 관리하는 역할을 합니다.

## `exec` 도구: 명령어 실행

쉘 명령어를 실행할 때 사용하는 핵심 도구입니다.

- **`yieldMs`** (기본값 10초): 명령어가 이 시간보다 오래 걸리면 자동으로 백그라운드로 전환됩니다.
- **`background`**: true로 설정하면 처음부터 백그라운드에서 즉시 실행을 시작합니다.
- **`timeout`** (기본값 1800초): 지정된 시간이 지나면 프로세스를 강제로 종료합니다.
- **`pty`**: 실제 터미널(TTY) 환경이 필요한 경우 사용합니다.

### 실행 동작
- 짧은 작업은 결과를 즉시 반환합니다.
- 백그라운드로 전환된 작업은 `status: "running"` 상태와 함께 `sessionId`를 반환합니다. 출력 내용은 메모리에 보관되며, 나중에 `process` 도구로 확인해야 합니다.

## `process` 도구: 실행 관리

백그라운드에서 돌아가는 세션들을 제어합니다.

- **`list`**: 현재 실행 중이거나 이미 종료된 백그라운드 세션 목록을 확인합니다.
- **`poll`**: 특정 세션의 새로운 출력 내용을 가져옵니다. 프로세스가 종료되었는지도 함께 확인합니다.
- **`log`**: 지금까지의 전체 출력 로그를 읽어옵니다. (행 단위 `offset`, `limit` 지원)
- **`write`**: 실행 중인 프로세스에 입력(`stdin`)을 보냅니다. (예: "y/n" 응답 등)
- **`kill`**: 실행 중인 세션을 강제로 종료합니다.
- **`clear`**: 종료된 세션 정보를 메모리에서 삭제합니다.

## 주의 사항 및 설정

- **휘발성**: 백그라운드 세션은 게이트웨이 프로세스가 재시작되면 모두 사라집니다 (디스크에 저장되지 않음).
- **에이전트 범위**: 각 에이전트는 자기가 시작한 프로세스 세션만 볼 수 있습니다.
- **청소**: 종료된 세션은 일정 시간(기본 30분)이 지나면 메모리 절약을 위해 자동으로 삭제됩니다 (`tools.exec.cleanupMs`).
- **알림**: 백그라운드 작업이 완료되면 시스템 이벤트를 통해 사용자에게 알림을 보낼 수 있습니다 (`tools.exec.notifyOnExit`).

상세한 매개변수나 환경 변수 설정은 [영문 원본 문서](/gateway/background-process)를 참고하십시오.
