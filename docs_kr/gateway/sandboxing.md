# 샌드박싱 (Sandboxing)

OpenClaw는 모델이 실행하는 도구들의 영향을 최소화하기 위해 **도커(Docker) 컨테이너 안에서 도구를 실행**하는 기능을 제공합니다. 이는 선택 사항이며, 비활성화된 경우 도구는 호스트 시스템에서 직접 실행됩니다.

샌드박싱은 완벽한 보안 경계는 아니지만, 모델이 실수로 시스템 파일을 삭제하거나 위험한 프로세스를 실행하는 것을 효과적으로 차단합니다.

## 샌드박싱 대상

- **도구 실행**: `exec`, `read`, `write`, `edit`, `process` 등 모든 도구 실행.
- **브라우저**: 별도의 격리된 브라우저 컨테이너에서 인터넷 서핑 등을 수행합니다.

**샌드박싱되지 않는 항목:**
- 게이트웨이 프로세스 자체.
- **상승된 권한(Elevated mode)**으로 실행되는 도구. (호스트에서 직접 실행됨)

## 주요 설정 모드

`agents.defaults.sandbox.mode`를 통해 제어합니다:
- **`off`**: 샌드박싱을 사용하지 않습니다.
- **`non-main`**: 특정 채널이나 그룹 채팅 등 "비주요(non-main)" 세션만 샌드박싱합니다. (기본 추천 설정)
- **`all`**: 모든 대화 세션을 샌드박스에서 실행합니다.

## 워크스페이스 접근 권한

샌드박스가 호스트의 파일을 얼마나 볼 수 있는지 결정합니다:
- **`none`**: 호스트 파일을 전혀 보지 못하며, 샌드박스 전용 임시 공간만 사용합니다.
- **`ro`**: 워크스페이스를 **읽기 전용**으로 연결합니다. (파일 수정 불가)
- **`rw`**: 워크스페이스를 **읽고 쓰기** 가능하도록 연결합니다.

## 시작하기

1. **이미지 빌드**: 처음 한 번은 샌드박스 이미지를 빌드해야 합니다.
   ```bash
   scripts/sandbox-setup.sh
   # 브라우저 샌드박스가 필요한 경우
   scripts/sandbox-browser-setup.sh
   ```
2. **네트워크 설정**: 기본적으로 샌드박스는 인터넷 연결이 차단되어 있습니다 (`network: "none"`). 인터넷 연결이 필요하다면 `bridge` 등으로 변경해야 합니다.

## 요약 설정 예시 (`openclaw.json`)

```json5
{
  "agents": {
    "defaults": {
      "sandbox": {
        "mode": "non-main", // 그룹 채팅 등은 격리 실행
        "scope": "session", // 대화방마다 별도의 컨테이너 생성
        "workspaceAccess": "ro" // 워크스페이스는 읽기만 허용
      }
    }
  }
}
```

## 도움말 및 진단
현재 내 세션이 샌드박스에서 돌아가고 있는지, 어떤 권한이 적용되어 있는지 확인하려면 다음 명령어를 사용하세요:
```bash
openclaw sandbox explain
```

상세한 보안 정책과 고급 바인드 마운트 설정은 [영문 원본 문서](/gateway/sandboxing)를 참고하십시오.
