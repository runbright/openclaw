# 샌드박스 vs 도구 정책 vs 권한 상승

OpenClaw에는 서로 관련되어 있지만 다른 세 가지 제어 계층이 있습니다:

1. **샌드박스 (Sandbox)**: 도구가 **어디서 실행될지** 결정합니다 (Docker 컨테이너 vs 호스트 머신).
2. **도구 정책 (Tool policy)**: **어떤 도구를 사용할 수 있는지** 결정합니다.
3. **권한 상승 (Elevated)**: 샌드박스 상태일 때 호스트에서 명령을 실행할 수 있도록 해주는 **exec 전용 탈출구**입니다.

## 빠른 디버깅

OpenClaw가 현재 어떻게 작동하고 있는지 확인하려면 다음 명령어를 사용하세요:

```bash
openclaw sandbox explain
openclaw sandbox explain --agent 에이전트ID
```

이 명령은 다음 정보를 출력합니다:
- 현재 적용된 샌드박스 모드/범위/워크스페이스 권한
- 해당 세션이 현재 샌드박스 내에서 작동 중인지 여부
- 유효한 도구 허용/차단 목록과 그 설정의 출처(에이전트별 설정, 전역 설정 등)
- 권한 상승 제한 상태 및 수정해야 할 설정 키 경로

## 1. 샌드박스: 도구가 실행되는 위치

`agents.defaults.sandbox.mode` 설정으로 제어합니다:
- `"off"`: 모든 작업이 호스트 머신에서 실행됩니다.
- `"non-main"`: 메인(main) 세션이 아닌 경우(예: 단체방, 채널 등)에만 샌드박스를 사용합니다.
- `"all"`: 모든 작업이 샌드박스 내에서 실행됩니다.

## 2. 도구 정책: 사용 가능한 도구 결정

다음 레이어들이 순서대로 적용됩니다:
- **전역/에이전트별 정책**: `tools.allow`/`tools.deny` 및 에이전트별 `tools.allow`/`deny`
- **샌드박스 전용 정책** (샌드박스 안에서만 적용): `tools.sandbox.tools.allow`/`deny`

**규칙**:
- 차단(`deny`) 목록에 있으면 항상 차단됩니다.
- 허용(`allow`) 목록이 비어 있지 않다면, 그 목록에 없는 모든 도구는 차단된 것으로 간주합니다.
- 도구 정책은 가장 강력한 제한입니다. `/exec` 명령으로도 정책상 차단된 도구를 사용할 수는 없습니다.

### 도구 그룹 (Shorthands)
정책 설정 시 여러 도구를 묶은 그룹 이름을 사용할 수 있습니다:
- `group:runtime`: `exec`, `bash`, `process`
- `group:fs`: `read`, `write`, `edit`, `apply_patch` (파일 시스템 관련)
- `group:ui`: `browser`, `canvas`
- `group:memory`: `memory_search`, `memory_get`

## 3. 권한 상승: 호스트에서 명령 실행 (exec 전용)

권한 상승(Elevated)은 새로운 도구를 허용해 주는 것이 아니라, 오직 `exec` 도구의 실행 위치에만 영향을 줍니다.

- 샌드박스 환경일 때 `/elevated on`을 사용하면 명령이 호스트에서 실행됩니다.
- 이미 호스트에서 실행 중이라면(샌드박스 오프), 이 설정은 별다른 영향이 없습니다.
- 권한 상승은 발신자 허용 목록(`tools.elevated.allowFrom`)에 있는 사용자만 사용할 수 있습니다.

상세 내용: [권한 상승 모드](/tools/elevated)

## 일반적인 "샌드박스 감옥" 해결 방법

### "도구가 샌드박스 도구 정책에 의해 차단되었습니다"
- **방법 1**: 샌드박스 자체를 끕니다 (`sandbox.mode=off`).
- **방법 2**: 샌드박스 내에서 해당 도구를 허용합니다 (`tools.sandbox.tools.allow` 목록에 추가하거나 `deny` 목록에서 제거).

### "메인 세션인 줄 알았는데 왜 샌드박스에서 실행되나요?"
- `"non-main"` 모드에서는 그룹 채팅이나 특정 채널 채팅은 '메인'이 아닌 것으로 간주됩니다. 의도한 대로 작동하게 하려면 모드를 `"off"`로 바꾸거나 `sandbox explain`으로 현재 세션 키를 확인해 보세요.
