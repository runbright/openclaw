# 하트비트 (Heartbeat)

> **하트비트 vs 크론(Cron)?** 언제 무엇을 사용해야 할지는 [크론 vs 하트비트 비교 가이드](/automation/cron-vs-heartbeat)를 참고하십시오.

하트비트는 메인 세션에서 **주기적으로 에이전트를 깨워** 특별히 신경 써야 할 일이 있는지 스스로 확인하게 하는 기능입니다. 이를 통해 에이전트는 사용자가 먼저 말을 걸지 않아도 중요한 사항을 사용자에게 알릴 수 있습니다.

## 시작하기

1. 하트비트는 기본적으로 활성화되어 있습니다 (기본 주기: 30분). 필요에 따라 주기를 조정하세요.
2. 에이전트 워크스페이스에 `HEARTBEAT.md` 파일을 만들어 에이전트가 확인해야 할 체크리스트를 작성하세요 (권장).
3. 답변이 어디로 전달될지 결정하세요 (기본값: `target: "last"`, 마지막으로 대화한 채널).

### 설정 예시 (`openclaw.json`)

```json5
{
  "agents": {
    "defaults": {
      "heartbeat": {
        "every": "30m", // 30분마다 실행
        "target": "last", // 마지막 대화 채널로 알림 전송
        "activeHours": { "start": "08:00", "end": "22:00" } // 활동 시간 제한 (옵션)
      }
    }
  }
}
```

## 동작 원리 및 규칙

- **프롬프트**: 에이전트는 설정된 `prompt` 내용(또는 기본값)을 사용자 메시지로 인식하여 실행됩니다. 기본 프롬프트는 `HEARTBEAT.md` 파일을 읽고 지시사항을 따르라는 내용을 포함합니다.
- **응답 규칙**:
  - 에이전트가 할 일이 없다고 판단하여 **`HEARTBEAT_OK`**라고 답변하면, 이 메시지는 사용자에게 전달되지 않고 조용히 처리됩니다.
  - 중요한 알림이 있는 경우에만 에이전트가 작성한 텍스트가 사용자에게 전달됩니다.
- **활동 시간 (`activeHours`)**: 밤늦은 시간이나 이른 새벽에 알림이 오는 것을 방지하기 위해 활동 시간을 지정할 수 있습니다. 설정된 타임존을 기준으로 작동합니다.

## `HEARTBEAT.md` 활용 (체크리스트)

워크스페이스에 `HEARTBEAT.md` 파일을 두면 에이전트가 30분마다 무엇을 확인해야 할지 가이드라인을 줄 수 있습니다.

**예시 `HEARTBEAT.md`:**
```md
# 하트비트 체크리스트
- 읽지 않은 중요 메일이나 일정이 있는지 확인해줘.
- 진행 중인 작업 중 막힌 부분이 있다면 메모해두고 나중에 알려줘.
- 업무 시간 중이라면 가벼운 안 인사를 건네도 좋아.
```

## 주요 설정 옵션

- **`every`**: 하트비트 간격 (예: `30m`, `1h`, `0m`은 비활성화).
- **`model`**: 하트비트 실행 시 사용할 특정 모델 지정.
- **`target`**: 알림을 보낼 채널 (`last`, `whatsapp`, `telegram`, `none` 등).
- **`to`**: 특정 수신인 번호나 그룹 아이디로 강제 지정.
- **`includeReasoning`**: 에이전트가 왜 이 시점에 알림을 보냈는지 추론 과정(`Reasoning:`)을 함께 보고 싶을 때 사용합니다.

상세한 필드 사양이나 고급 설정 패턴은 [영문 원본 문서](/gateway/heartbeat)를 참고하십시오.
