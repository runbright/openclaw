# 닥터 (Doctor)

`openclaw doctor`는 OpenClaw의 종합 진단 및 수리 도구입니다. 잘못된 설정이나 상태를 바로잡고, 시스템의 전반적인 건강 상태를 확인하며, 즉시 실행 가능한 복구 단계를 제공합니다.

## 빠른 시작

```bash
# 시스템 진단 및 대화형 복구 시작
openclaw doctor

# 모든 질문에 '예'로 응답하여 자동 복구 (자동화 환경 권장)
openclaw doctor --yes

# 추천하는 복구 항목만 즉시 적용
openclaw doctor --repair
```

## 주요 기능

1. **설정 정규화 및 마이그레이션**: 이전 버전의 설정 형식을 현재 스키마에 맞게 자동으로 변환합니다.
2. **상태 무결성 점검**: 세션 저장소, 대화 기록 파일, 권한 설정 등이 올바른지 확인하고 문제를 해결합니다.
3. **모델 인증 상태 확인**: OAuth 토큰 만료 여부를 점검하고, 필요한 경우 자동으로 갱신하거나 재로그인을 안내합니다.
4. **서비스 엔진 진단**: 게이트웨이가 올바른 Node.js 버전에서 실행 중인지, 포트 충돌은 없는지, 시스템 서비스(launchd/systemd)가 정상인지 확인합니다.
5. **보안 경고**: 누구나 접근 가능한 DM 정책이나 누락된 게이트웨이 보안 토큰 등을 감지하여 경고합니다.
6. **샌드박스 복구**: 도커 기반 샌드박싱 기능을 사용하는 경우 이미지 존재 여부와 정상 작동을 점검합니다.

## 상세 동작 방식

### 설정 및 데이터 마이그레이션
OpenClaw는 버전 업데이트 시 데이터 구조가 바뀔 수 있습니다. `doctor`는 다음 항목들을 안전하게 옮겨줍니다:
- 세션 기록 및 트랜스크립트 경로 (`~/.openclaw/sessions/` → `~/.openclaw/agents/default/sessions/`)
- 에이전트 인증 정보 및 모델 설정 파일
- WhatsApp 인증 상태 데이터 정보

### 시스템 서비스 관리
백그라운드에서 실행되는 게이트웨이 서비스 파일을 검사하여 최적의 설정(재시작 지연 시간, 네트워크 대기 설정 등)으로 업데이트할 것을 권장합니다.

### 상태 디렉토리 보호
OpenClaw의 두뇌 역할을 하는 데이터 디렉토리(`~/.openclaw`)의 유실이나 권한 오류를 방지합니다. 파일 권한이 너무 느슨한 경우(예: 다른 사용자가 읽을 수 있는 경우) 보안을 위해 권한을 강화(`600`)할 것을 제안합니다.

## 트러블슈팅 팁

- **게이트웨이가 시작되지 않을 때**: `openclaw doctor`를 실행하여 출력되는 에러 메시지를 확인하세요. 대부분의 설정 오류는 닥터가 제안하는 수리 단계를 통해 해결됩니다.
- **인증 오류 발생 시**: 모델 제공자의 토큰이 만료되었을 수 있습니다. 닥터는 이를 실시간으로 감지하여 `claude setup-token` 실행 등을 안내합니다.
- **포트 충돌(`18789`)**: 이미 다른 게이트웨이 프로세스가 실행 중이거나 SSH 터널이 해당 포트를 점유하고 있는지 진단합니다.

상세한 마이그레이션 목록이나 기술적인 질문은 [영문 원본 문서](/gateway/doctor)를 참고하십시오.
