# 로그 (Logging)

OpenClaw는 두 가지 방식으로 로그를 관리합니다:
1. **콘솔 출력**: 터미널이나 디버그 UI에서 실시간으로 보는 로그.
2. **파일 로그**: 게이트웨이가 파일에 저장하는 상세한 JSON 기반 로그.

## 파일 기반 로그

- **저장 위치**: 기본적으로 `/tmp/openclaw/` 폴더에 `openclaw-YYYY-MM-DD.log` 형식으로 일자별로 저장됩니다.
- **설정**: `logging.file`로 경로를, `logging.level`로 로그 수준(`info`, `debug`, `trace` 등)을 조절할 수 있습니다.
- **확인 방법**:
  ```bash
  openclaw logs --follow  # 실시간으로 로그 파일 추적
  ```

> **주의**: `--verbose` 옵션은 콘솔에 보여지는 정보량만 늘릴 뿐, 로그 파일에 저장되는 수준은 바꾸지 않습니다. 상세한 내용을 파일에 남기려면 `logging.level`을 `debug`로 설정하세요.

## 콘솔 로그 스타일

터미널 출력 시 가독성을 위해 다양한 스타일을 지원합니다:
- **`pretty`**: 색상과 아이콘을 가미한 읽기 편한 형식 (기본값).
- **`compact`**: 정보를 압축하여 공간을 절약한 형식.
- **`json`**: 기계가 읽기 편하도록 모든 줄을 JSON 형식으로 출력.

## 민감 정보 가리기 (Redaction)

콘솔 로그에 API 키나 비밀 토큰이 그대로 노출되는 것을 방지합니다.
- **기본값**: `logging.redactSensitive: "tools"` (도구 실행 시의 토큰 등을 가림).
- **작동**: 길이가 긴 키는 앞 6자리와 뒤 4자리만 남기고 `***`로 표시합니다.

## WebSocket (WS) 통신 로그

게이트웨이는 WebSocket 통신 내용을 다음 두 모드로 출력합니다:
- **일반 모드**: 에러, 응답이 느린(50ms 이상) 호출, 파싱 에러만 간단히 출력합니다.
- **상세 모드 (`--verbose`)**: 모든 WebSocket 주고받기 데이터를 출력합니다.
  - `--ws-log compact`: 요청과 응답을 짝지어 깔끔하게 표시.
  - `--ws-log full`: 모든 프레임 메타데이터를 포함한 전체 표시.

## 주요 특징

- **하위 시스템 구분**: 로그 줄마다 `[gateway]`, `[whatsapp]` 등 해당 로그를 발생시킨 시스템 이름이 붙어 있어 파악이 쉽습니다.
- **색상 지원**: TTY를 감지하여 색상을 입혀 보여줍니다 (`NO_COLOR` 환경 변수 존중).
- **디버그 정보**: WhatsApp 메시지 본문 등 상세한 데이터는 `debug` 수준으로 분류되어 있으며, `--verbose` 옵션을 주었을 때만 콘솔에 나타납니다.

상세한 설정 필드나 정규식 패턴 지정 방법은 [영문 원본 문서](/gateway/logging)를 참고하십시오.
