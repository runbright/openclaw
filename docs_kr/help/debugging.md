# 디버깅 (Debugging)

OpenClaw 시스템을 개발하거나 복잡한 오류를 추적할 때 유용한 도구들을 설명합니다.

## 실시간 코드 감시 (`watch` 모드)

코드 수정 사항을 실시간으로 반영하여 게이트웨이를 다시 시작하려면 다음 명령어를 사용하세요:

```bash
pnpm gateway:watch
```

이 명령어는 `src` 폴더 내의 파일 변경을 감지하여 게이트웨이를 자동으로 재시작합니다. 추가적인 CLI 플래그를 뒤에 붙여서 실행할 수 있습니다.

## 개발 모드 및 프로필 분리 (`--dev`)

개발 중에는 기존 설정이나 데이터를 건드리지 않도록 분리된 환경을 사용하는 것이 좋습니다.

```bash
pnpm gateway:dev
```

**이 명령어가 수행하는 작업:**
- 상태 저장 디렉토리를 `~/.openclaw-dev`로 분리합니다.
- 게이트웨이 포트를 `19001`로 변경하여 기존 서비스와 충돌을 방지합니다.
- 기본 설정과 워크스페이스 파일을 자동으로 생성합니다.
- 에이전트의 페르소나를 'C3-PO'로 설정합니다.

초기 상태로 완전히 되돌리려면:
```bash
pnpm gateway:dev:reset
```

## 실시간 설정 변경 (`/debug`)

채팅창에서 바로 설정을 임시로 바꿔볼 수 있습니다. (메모리에서만 유지되며 파일은 저장되지 않습니다.)

- `/debug show`: 현재 적용된 임시 설정들을 보여줍니다.
- `/debug set <경로>=<값>`: 특정 설정값을 변경합니다.
- `/debug reset`: 모든 임시 설정을 초기화하고 파일 설정으로 돌아갑니다.

## 원본 스트림 로그 확인

모델이 보내주는 필터링 되지 않은 원본 응답을 확인하고 싶다면 다음 옵션을 사용하세요:

```bash
pnpm gateway:watch --raw-stream
```

로그 파일 위치: `~/.openclaw/logs/raw-stream.jsonl`

> **주의:** 원본 스트림 로그에는 개인 정보나 API 키가 포함될 수 있으므로, 디버깅 후에는 반드시 로그 파일을 삭제해 주세요.
