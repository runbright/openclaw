# 노드 (Nodes)

**노드**는 메인 게이트웨이에 연결되는 '보조 기기'(macOS, iOS, Android, 혹은 화면이 없는 컴퓨터 등)를 의미합니다. 노드를 연결하면 에이전트가 해당 기기의 카메라를 보거나, 화면을 캡처하고, 시스템 명령어를 실행하는 등의 작업을 수행할 수 있습니다.

## 노드 연결 및 승인 (Pairing)

노드를 게이트웨이에 처음 연결하면 보안을 위해 승인 절차가 필요합니다.

1.  노드 기기에서 게이트웨이 주소로 접속을 시도합니다.
2.  게이트웨이 터미널에서 다음 명령어를 입력하여 대기 중인 요청을 확인합니다.
    ```bash
    openclaw devices list
    ```
3.  요청 ID를 확인한 후 승인합니다.
    ```bash
    openclaw devices approve <요청ID>
    ```

## 주요 노드 기능

### 1. 화면 및 캔버스 제어 (`canvas`)
에이전트가 웹 페이지를 열거나 현재 화면을 스크린샷 찍을 수 있습니다.
- `openclaw nodes canvas snapshot`: 노드의 현재 화면을 캡처합니다.
- `openclaw nodes canvas navigate <URL>`: 특정 웹 사이트로 이동합니다.

### 2. 카메라 및 영상 (`camera`)
노드의 카메라를 조작하여 사진을 찍거나 짧은 영상을 녹화합니다.
- `openclaw nodes camera snap`: 사진 촬영 (전면/후면).
- `openclaw nodes camera clip --duration 10s`: 10초 동안 영상 녹화.

### 3. 시스템 명령어 실행 (`system.run`)
다른 컴퓨터(노드)에서 쉘 명령어를 실행할 수 있습니다. 예를 들어, 리눅스 서버를 게이트웨이로 쓰고 내 맥북을 노드로 연결하여 맥북의 파일을 정리하게 할 수 있습니다.

### 4. 기타 기능
- **위치 (`location`)**: 노드의 현재 GPS 위치를 확인합니다.
- **SMS (Android)**: 안드로이드 노드를 통해 문자 메시지를 보낼 수 있습니다.
- **알림 (`notify`)**: 노드 기기에 푸시 알림을 보냅니다.

## 노드 호스트 (Node Host)

별도의 UI 없이 터미널에서만 작동하는 노드를 만들고 싶다면 다음 명령어를 사용하세요:

```bash
openclaw node run --host <게이트웨이_주소> --display-name "나의서버"
```

## 주의 사항
- `canvas`나 `camera` 기능을 사용하려면 노드 앱이 **포그라운드(화면에 띄워진 상태)**여야 합니다.
- 모든 명령어 실행은 보안을 위해 **승인 목록(Allowlist)**에 등록된 것만 가능하도록 설정할 수 있습니다.
