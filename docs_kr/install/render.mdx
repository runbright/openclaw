---
title: Render에 배포
---

Infrastructure as Code를 사용하여 Render에 OpenClaw을 배포합니다. 포함된 `render.yaml` Blueprint가 서비스, 디스크, 환경 변수 등 전체 스택을 선언적으로 정의하므로, 원클릭으로 배포하고 코드와 함께 인프라를 버전 관리할 수 있습니다.

## 사전 요구 사항

- [Render 계정](https://render.com) (무료 티어 사용 가능)
- 선호하는 [모델 프로바이더](/providers)의 API 키

## Render Blueprint로 배포

[Deploy to Render](https://render.com/deploy?repo=https://github.com/openclaw/openclaw)

이 링크를 클릭하면:

1. 저장소 루트의 `render.yaml` Blueprint에서 새 Render 서비스를 생성합니다.
2. `SETUP_PASSWORD` 설정을 요청합니다
3. Docker 이미지를 빌드하고 배포합니다

배포되면 서비스 URL은 `https://<service-name>.onrender.com` 패턴을 따릅니다.

## Blueprint 이해

Render Blueprint는 인프라를 정의하는 YAML 파일입니다. 이 저장소의 `render.yaml`은
OpenClaw 실행에 필요한 모든 것을 구성합니다:

```yaml
services:
  - type: web
    name: openclaw
    runtime: docker
    plan: starter
    healthCheckPath: /health
    envVars:
      - key: PORT
        value: "8080"
      - key: SETUP_PASSWORD
        sync: false # 배포 시 프롬프트
      - key: OPENCLAW_STATE_DIR
        value: /data/.openclaw
      - key: OPENCLAW_WORKSPACE_DIR
        value: /data/workspace
      - key: OPENCLAW_GATEWAY_TOKEN
        generateValue: true # 보안 토큰 자동 생성
    disk:
      name: openclaw-data
      mountPath: /data
      sizeGB: 1
```

사용된 주요 Blueprint 기능:

| 기능                  | 용도                                                       |
| --------------------- | ---------------------------------------------------------- |
| `runtime: docker`     | 저장소의 Dockerfile에서 빌드                                |
| `healthCheckPath`     | Render가 `/health`를 모니터링하고 비정상 인스턴스를 재시작   |
| `sync: false`         | 배포 시 값을 프롬프트 (시크릿)                               |
| `generateValue: true` | 암호학적으로 안전한 값을 자동 생성                            |
| `disk`                | 재배포에도 유지되는 영구 스토리지                              |

## 플랜 선택

| 플랜      | 스핀다운          | 디스크        | 적합한 용도                    |
| --------- | ----------------- | ------------- | ------------------------------ |
| Free      | 15분 비활성 후    | 사용 불가     | 테스트, 데모                   |
| Starter   | 하지 않음         | 1GB+          | 개인 사용, 소규모 팀           |
| Standard+ | 하지 않음         | 1GB+          | 프로덕션, 다중 채널            |

Blueprint는 기본적으로 `starter`입니다. 무료 티어를 사용하려면 포크의 `render.yaml`에서
`plan: free`로 변경하세요 (단, 영구 디스크가 없으므로 배포마다 설정이 초기화됩니다).

## 배포 후

### 설정 마법사 완료

1. `https://<your-service>.onrender.com/setup`으로 이동
2. `SETUP_PASSWORD` 입력
3. 모델 프로바이더를 선택하고 API 키를 붙여넣기
4. 선택적으로 메시징 채널 구성 (Telegram, Discord, Slack)
5. **Run setup** 클릭

### Control UI 접근

웹 대시보드는 `https://<your-service>.onrender.com/openclaw`에서 사용할 수 있습니다.

## Render 대시보드 기능

### 로그

**Dashboard → 서비스 → Logs**에서 실시간 로그를 확인합니다. 다음으로 필터링:

- 빌드 로그 (Docker 이미지 생성)
- 배포 로그 (서비스 시작)
- 런타임 로그 (애플리케이션 출력)

### 셸 접근

디버깅을 위해 **Dashboard → 서비스 → Shell**에서 셸 세션을 엽니다. 영구 디스크는 `/data`에 마운트됩니다.

### 환경 변수

**Dashboard → 서비스 → Environment**에서 변수를 수정합니다. 변경하면 자동 재배포가 트리거됩니다.

### 자동 배포

원본 OpenClaw 저장소를 사용하는 경우, Render는 OpenClaw을 자동 배포하지 않습니다. 업데이트하려면 대시보드에서 수동 Blueprint 동기화를 실행하세요.

## 커스텀 도메인

1. **Dashboard → 서비스 → Settings → Custom Domains**로 이동
2. 도메인 추가
3. 안내에 따라 DNS 구성 (`*.onrender.com`에 대한 CNAME)
4. Render가 TLS 인증서를 자동으로 프로비저닝

## 확장

Render는 수평 및 수직 확장을 지원합니다:

- **수직**: 플랜을 변경하여 더 많은 CPU/RAM 확보
- **수평**: 인스턴스 수 증가 (Standard 플랜 이상)

OpenClaw의 경우 수직 확장으로 보통 충분합니다. 수평 확장에는 스티키 세션 또는 외부 상태 관리가 필요합니다.

## 백업 및 마이그레이션

언제든지 설정과 워크스페이스를 내보내세요:

```
https://<your-service>.onrender.com/setup/export
```

모든 OpenClaw 호스트에서 복원할 수 있는 이식 가능한 백업을 다운로드합니다.

## 문제 해결

### 서비스가 시작되지 않음

Render 대시보드에서 배포 로그를 확인하세요. 일반적인 문제:

- `SETUP_PASSWORD` 누락 — Blueprint가 이를 프롬프트하지만, 설정되어 있는지 확인하세요
- 포트 불일치 — `PORT=8080`이 Dockerfile의 노출 포트와 일치하는지 확인

### 느린 콜드 스타트 (무료 티어)

무료 티어 서비스는 15분 비활성 후 스핀다운됩니다. 스핀다운 후 첫 번째 요청은 컨테이너가 시작되는 동안 몇 초가 걸립니다. 상시 가동을 위해 Starter 플랜으로 업그레이드하세요.

### 재배포 후 데이터 손실

무료 티어 (영구 디스크 없음)에서 발생합니다. 유료 플랜으로 업그레이드하거나,
`/setup/export`를 통해 정기적으로 설정을 내보내세요.

### 헬스 체크 실패

Render는 30초 내에 `/health`에서 200 응답을 기대합니다. 빌드는 성공하지만 배포가 실패하면, 서비스가 시작하는 데 너무 오래 걸리는 것일 수 있습니다. 확인:

- 빌드 로그에 오류가 있는지
- `docker build && docker run`으로 컨테이너가 로컬에서 실행되는지
