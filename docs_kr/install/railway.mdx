---
title: Railway에 배포
---

원클릭 템플릿으로 Railway에 OpenClaw을 배포하고 브라우저에서 설정을 완료하세요.
이것은 "서버에서 터미널 없이" 가장 쉬운 경로입니다: Railway가 게이트웨이를 실행하고,
`/setup` 웹 마법사를 통해 모든 것을 구성합니다.

## 빠른 체크리스트 (신규 사용자)

1. **Deploy on Railway** (아래)를 클릭합니다.
2. `/data`에 마운트된 **Volume**을 추가합니다.
3. 필수 **Variables**를 설정합니다 (최소 `SETUP_PASSWORD`).
4. 포트 `8080`에서 **HTTP Proxy**를 활성화합니다.
5. `https://<your-railway-domain>/setup`을 열고 마법사를 완료합니다.

## 원클릭 배포

<a href="https://railway.com/deploy/clawdbot-railway-template" target="_blank" rel="noreferrer">
  Deploy on Railway
</a>

배포 후 **Railway → 서비스 → Settings → Domains**에서 공개 URL을 찾으세요.

Railway는 다음 중 하나를 제공합니다:

- 생성된 도메인 (보통 `https://<something>.up.railway.app`), 또는
- 연결한 커스텀 도메인.

그런 다음 열기:

- `https://<your-railway-domain>/setup` — 설정 마법사 (비밀번호 보호)
- `https://<your-railway-domain>/openclaw` — Control UI

## 제공되는 기능

- 호스팅된 OpenClaw 게이트웨이 + Control UI
- `/setup`의 웹 설정 마법사 (터미널 명령 불필요)
- Railway Volume (`/data`)을 통한 영구 스토리지로 설정/자격 증명/워크스페이스가 재배포에도 유지
- `/setup/export`에서 백업 내보내기로 나중에 Railway에서 마이그레이션 가능

## 필수 Railway 설정

### 공개 네트워킹

서비스에 **HTTP Proxy**를 활성화합니다.

- 포트: `8080`

### Volume (필수)

다음 경로에 마운트된 볼륨을 연결합니다:

- `/data`

### 변수

서비스에 다음 변수를 설정합니다:

- `SETUP_PASSWORD` (필수)
- `PORT=8080` (필수 — 공개 네트워킹의 포트와 일치해야 함)
- `OPENCLAW_STATE_DIR=/data/.openclaw` (권장)
- `OPENCLAW_WORKSPACE_DIR=/data/workspace` (권장)
- `OPENCLAW_GATEWAY_TOKEN` (권장; 관리자 시크릿으로 취급)

## 설정 흐름

1. `https://<your-railway-domain>/setup`을 방문하고 `SETUP_PASSWORD`를 입력합니다.
2. 모델/인증 프로바이더를 선택하고 키를 붙여넣습니다.
3. (선택) Telegram/Discord/Slack 토큰을 추가합니다.
4. **Run setup**을 클릭합니다.

Telegram DM이 페어링으로 설정된 경우, 설정 마법사에서 페어링 코드를 승인할 수 있습니다.

## 채팅 토큰 받기

### Telegram 봇 토큰

1. Telegram에서 `@BotFather`에게 메시지 보내기
2. `/newbot` 실행
3. 토큰 복사 (`123456789:AA...` 형태)
4. `/setup`에 붙여넣기

### Discord 봇 토큰

1. [https://discord.com/developers/applications](https://discord.com/developers/applications)으로 이동
2. **New Application** → 이름 선택
3. **Bot** → **Add Bot**
4. Bot → Privileged Gateway Intents에서 **MESSAGE CONTENT INTENT 활성화** (필수, 활성화하지 않으면 봇이 시작 시 충돌)
5. **Bot Token**을 복사하고 `/setup`에 붙여넣기
6. 봇을 서버에 초대 (OAuth2 URL Generator; scopes: `bot`, `applications.commands`)

## 백업 및 마이그레이션

다음에서 백업을 다운로드하세요:

- `https://<your-railway-domain>/setup/export`

설정이나 메모리를 잃지 않고 다른 호스트로 마이그레이션할 수 있도록 OpenClaw 상태 + 워크스페이스를 내보냅니다.
