# 안드로이드 앱 (Android Node)

안드로이드 앱은 OpenClaw의 **동반 노드(Companion Node)** 역할을 합니다. 안드로이드 기기 자체가 게이트웨이(서버)가 되지는 않으며, 이미 실행 중인 게이트웨이에 연결하여 기기의 카메라나 화면 기능을 공유합니다.

## 시스템 구성
- **역할**: 보조 기기(노드)
- **필수 사항**: macOS, Linux, 또는 Windows(WSL2)에서 실행 중인 OpenClaw 게이트웨이
- **연결 방식**: 동일한 Wi-Fi(LAN) 환경 혹은 Tailscale과 같은 VPN을 통한 연결

## 연결 가이드 (Runbook)

### 1단계: 게이트웨이 준비
먼저 서버 컴퓨터에서 오픈클로 게이트웨이를 실행합니다:
```bash
openclaw gateway status
```
상태가 `running`인 것을 확인하세요.

### 2단계: 안드로이드 앱에서 연결
1.  **OpenClaw 안드로이드 앱**을 엽니다.
2.  **Settings** 메뉴로 이동합니다.
3.  **Discovered Gateways** 목록에서 자신의 게이트웨이를 찾아 **Connect**를 누릅니다.
    - 목록에 보이지 않는다면 **Advanced → Manual Gateway**에서 직접 서버 IP와 포트(기본 18789)를 입력하세요.

### 3단계: 기기 승인 (Pairing)
보안을 위해 처음 연결할 때는 서버에서 승인이 필요합니다:
```bash
openclaw devices list
openclaw devices approve <요청ID>
```

## 주요 기능

- **채팅 (`Chat`)**: 서버의 대화 기록을 공유하며 어디서든 에이전트와 대화할 수 있습니다.
- **캔버스 (`Canvas`)**: 에이전트가 만든 웹 결과물을 휴대폰 화면에 띄우고 조작합니다.
- **카메라 (`Camera`)**: 에이전트가 휴대폰 카메라로 주변을 보고 분석하게 할 수 있습니다.
    - `camera.snap`: 사진 찍기
    - `camera.clip`: 짧은 영상 찍기

## 주의 사항
- `canvas`와 `camera` 기능은 안드로이드 앱이 **포그라운드(화면에 켜진 상태)**일 때만 작동합니다.
- 카메라 기능을 사용하려면 기기에서 카메라 및 마이크 권한을 허용해야 합니다.
