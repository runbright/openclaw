# iOS 앱 (iOS Node)

현재 iOS 앱은 내부 프리뷰 버전으로 제공되고 있습니다. 안드로이드와 마찬가지로 OpenClaw 게이트웨이에 연결하여 사용하는 **동반 노드(Companion Node)**입니다.

## 기능 개요
- **게이트웨이 연결**: 웹소켓을 통해 로컬 네트워크나 Tailscale 환경에서 게이트웨이에 연결됩니다.
- **노드 기능 제공**:
    - **캔버스 (Canvas)**: 에이전트가 생성한 UI나 대시보드를 표시합니다.
    - **카메라 및 스크린샷**: 현재 화면이나 카메라 이미지를 에이전트에게 보냅니다.
    - **위치 정보**: 기기의 현재 좌표를 공유합니다. (에이전트에게 허용한 경우)
    - **보이스 웨이크 (Voice Wake)**: 음성으로 에이전트를 호출합니다.

## 시스템 요구 사항
- 별도의 기기(macOS, Linux, WSL2)에서 실행 중인 **OpenClaw 게이트웨이**가 필요합니다.
- 동일한 네트워크에 있거나 VPN으로 연결되어 있어야 합니다.

## 빠른 시작 (연결 및 승인)

1.  **게이트웨이 확인**: 서버에서 게이트웨이가 실행 중인지 확인합니다.
2.  **연결 시도**: iOS 앱의 **Settings**에서 자동으로 검색된 게이트웨이를 선택하거나, **Manual Host**에 서버 주소를 입력합니다.
3.  **기기 승인**: 보안을 위해 서버에서 연결을 승인해 줍니다.
    ```bash
    openclaw devices list
    openclaw devices approve <요청ID>
    ```

## 주요 오류 및 해결 방법

- **`NODE_BACKGROUND_UNAVAILABLE`**: iOS 보안 정책상 카메라나 화면 캡처 기능은 앱이 **화면에 띄워져 있을 때(Foreground)**만 가능합니다.
- **페어링 요청이 뜨지 않음**: 서버 터미널에서 `openclaw nodes pending`을 입력하여 대기 대상을 직접 확인하세요.
- **앱 재설치 후 연결 안 됨**: 기존 인증 정보가 만료되었을 수 있습니다. 서버에서 기존 노드를 삭제하고 다시 페어링해 주세요.
