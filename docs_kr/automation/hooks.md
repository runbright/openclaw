# 후크 (Hooks)

후크는 에이전트의 명령어(예: `/new`)나 시스템 이벤트가 발생할 때 특정 스크립트를 자동으로 실행하도록 하는 시스템입니다.

## 주요 용도
- **세션 초기화 시 메모리 저장**: `/new` 명령어 실행 시, 이전 대화 맥락을 요약하여 메모리 파일에 저장합니다.
- **명령어 로깅**: 모든 명령어를 파일에 기록하여 추후 디버깅에 활용합니다.
- **자동 파일 생성**: 에이전트가 생성되거나 리셋될 때 필요한 기본 파일들을 워크스페이스에 자동으로 배치합니다.
- **부팅 스크립트 실행**: 게이트웨이가 시작될 때 특정 명령어(예: `BOOT.md`)를 실행합니다.

## 기본 제공 후크 (Bundled Hooks)

OpenClaw 설치 시 다음과 같은 유용한 후크들이 기본적으로 포함되어 있습니다:

- **💾 session-memory**: 세션을 리셋할 때 대화 내용을 메모리 폴더에 자동 저장합니다.
- **📝 command-logger**: 모든 명령어 이벤트를 로그 파일(`commands.log`)에 기록합니다.
- **📎 bootstrap-extra-files**: 워크스페이스를 처음 만들 때 추가 파일들을 자동으로 넣어줍니다.

## 후크 관리 명령어

터미널에서 다음 명령어를 통해 후크를 관리할 수 있습니다:

```bash
openclaw hooks list       # 사용 가능한 후크 목록 보기
openclaw hooks enable <이름>  # 특정 후크 활성화
openclaw hooks check      # 후크 작동 상태 확인
openclaw hooks info <이름>   # 후크 상세 정보 확인
```

## 후크가 저장되는 위치

1.  **워크스페이스 후크**: `<워크스페이스>/hooks/` (이 에이전트 전용)
2.  **사용자 후크**: `~/.openclaw/hooks/` (내 컴퓨터의 모든 프로젝트 공통)
3.  **내장 후크**: OpenClaw 프로그램 내부의 기본 후크

## 나만의 후크 만들기

후크는 `HOOK.md`(설명 파일)와 `handler.ts`(실행 로직 파일)가 들어있는 디렉토리 형태입니다. 간단한 TypeScript 지식만 있다면 누구나 OpenClaw의 기능을 확장할 수 있습니다.
