# 크론 vs 하트비트: 어떤 것을 사용해야 할까요?

OpenClaw에서는 예약된 작업을 수행하기 위해 **하트비트**와 **크론**이라는 두 가지 메커니즘을 제공합니다. 사용 사례에 맞는 최적의 선택을 도와드립니다.

## 빠른 의사결정 가이드

| 사용 사례 | 추천 방식 | 이유 |
| :--- | :--- | :--- |
| 30분마다 이메일 확인 | **하트비트** | 다른 체크 항목들과 묶어서 처리 가능 (비용 절감) |
| 매일 아침 9시 정각에 보고서 전송 | **크론 (격리 세션)** | 정확한 시간 준수가 필요함 |
| 일정표(캘린더) 모니터링 | **하트비트** | 주기적인 상황 인식에 자연스럽게 녹아듬 |
| 주간 딥 분석 (복잡한 작업) | **크론 (격리 세션)** | 독립된 환경에서 고성능 모델 사용 가능 |
| "20분 뒤에 알려줘" (리마인더) | **크론 (메인 세션)** | 1회성 정밀 예약 작업 |

## 하트비트 (Heartbeat): 주기적인 상황 인식

하트비트는 **메인 대화 세션**에서 정해진 간격(기본 30분)마다 실행됩니다. 에이전트가 "뭐 새로운 것 없나?" 하고 주변 상황을 살피는 용도입니다.

- **장점**: 여러 가지 체크 항목(이메일, 날씨, 일정 등)을 한 번의 AI 실행으로 묶어서 처리할 수 있어 비용이 저렴합니다.
- **특징**: 에이전트가 대화의 맥락을 알고 있으므로, 어떤 것이 시급한지 똑똑하게 판단할 수 있습니다.

## 크론 (Cron): 정밀한 스케줄링

크론 작업은 **정확한 시각**에 실행되며, 대화 맥락이 없는 **격리된 세션**에서 독립적으로 작동할 수도 있습니다.

- **장점**: 분 단위, 초 단위까지 정확한 실행 시간을 보장합니다.
- **특징**: 메인 대화창을 어지럽히지 않고 배경에서 조용히 분석 작업을 수행하거나, 특정 시간에만 고성능 모델을 사용하도록 설정할 수 있습니다.

## 결정 기준 (Flowchart)

1.  **정확한 시간이 중요한가?** -> 예: **크론**
2.  **독립된 환경(메인 대화와 별개)에서 실행되어야 하는가?** -> 예: **크론(격리 세션)**
3.  **여러 가지 주기적 체크 항목을 한 번에 처리하고 싶은가?** -> 예: **하트비트** (HEARTBEAT.md 사용)
4.  **1회성 알림인가?** -> 예: **크론 (`--at` 사용)**

## 두 기능의 조화로운 사용

가장 효율적인 방법은 두 기능을 **함께** 쓰는 것입니다.

- **하트비트**: 일상적인 모니터링(메일, 일정 확인 등) 담당. 30분마다 한 번씩 쓱 훑어봅니다.
- **크론**: 중요한 정기 보고서(매주 월요일 주간 결산 등)와 짧은 리마인더 담당.
