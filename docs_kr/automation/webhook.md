# 웹훅 (Webhooks)

OpenClaw 게이트웨이는 외부 시스템으로부터 HTTP 알림을 받아 특정 작업을 수행할 수 있는 웹훅 엔드포인트를 제공합니다.

## 활성화 설정
`openclaw.json` 파일에서 웹훅을 활성화하고 보안용 토큰을 설정합니다.

```json5
{
  "hooks": {
    "enabled": true,
    "token": "비밀_토큰_문자열", // 필수: 외부 요청 시 이 토큰이 필요합니다.
    "path": "/hooks" // 기본 엔드포인트 경로
  }
}
```

## 인증 방식 (Auth)
모든 웹훅 요청은 헤더에 토큰을 포함해야 합니다.
- **Header**: `Authorization: Bearer <토큰>` 또는 `x-openclaw-token: <토큰>`

## 주요 엔드포인트

### 1. `/hooks/wake` (에이전트 깨우기)
에이전트에게 시스템 메시지를 전달하고 즉시 작업을 시작하도록 합니다.
- **데이터 예시**: `{ "text": "새로운 이메일이 도착했습니다." }`

### 2. `/hooks/agent` (격리된 에이전트 실행)
메인 대화와는 별개로 특정 작업을 처리하는 에이전트를 실행합니다.
- **데이터 예시**:
```json
{
  "message": "이 메일 내용을 요약해줘",
  "deliver": true, // 결과를 채팅 채널로 전송할지 여부
  "channel": "whatsapp",
  "to": "+821012345678"
}
```

## 매핑 및 변환 (Mappings & Transforms)
외부 서비스마다 보내는 데이터 형식이 다르기 때문에, 이를 OpenClaw가 이해할 수 있는 형식으로 변환하는 기능입니다.

- **프리셋 (`presets`)**: `["gmail"]` 처럼 자주 쓰이는 서비스의 설정을 불러올 수 있습니다.
- **커스텀 매핑 (`mappings`)**: 특정 경로로 들어온 데이터를 어떤 에이전트에게 보낼지, 어떤 메시지 템플릿을 쓸지 설정합니다.
- **TypeScript 변환 (`transforms`)**: 복잡한 데이터 처리가 필요한 경우 직접 스크립트 파일을 작성하여 데이터를 가공할 수 있습니다.
