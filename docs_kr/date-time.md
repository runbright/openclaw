# 날짜 및 시간 (Date & Time)

OpenClaw는 기본적으로 **전송 타임스탬프에 호스트(서버)의 로컬 시간**을 사용하며, **시스템 프롬프트에는 사용자의 시간대**를 적용합니다.

## 메시지 봉투 (기본값: 로컬 시간)

수신된 메시지는 분 단위의 정밀도를 가진 타임스탬프로 감싸집니다:

```
[Provider ... 2026-01-05 16:26 PST] 메시지 내용
```

이 시간은 기본적으로 게이트웨이가 실행 중인 컴퓨터의 로컬 시간을 따릅니다. 설정을 통해 이를 변경할 수 있습니다:

```json5
{
  "agents": {
    "defaults": {
      "envelopeTimezone": "local", // "utc", "local", "user", 또는 IANA 시간대(예: "Asia/Seoul")
      "envelopeTimestamp": "on", // 절대 시간 표시 여부
      "envelopeElapsed": "on" // 경과 시간 표시 여부 (예: +2m)
    }
  }
}
```

- `user`로 설정하면 `agents.defaults.userTimezone` 값을 사용합니다.
- `envelopeElapsed`를 켜면 이전 메시지로부터 얼마나 지났는지(+30s 등)를 함께 표시합니다.

## 시스템 프롬프트의 시간 정보

에이전트가 현재 시간을 정확히 알 수 있도록 시스템 프롬프트에 시간대 정보를 포함합니다:

```
Time zone: Asia/Seoul
```

에이전트가 정확한 시각을 알아야 하는 경우, 에이전트는 `session_status` 도구를 사용하여 현재 시각을 확인합니다.

## 사용자 시간대 및 형식 설정

```json5
{
  "agents": {
    "defaults": {
      "userTimezone": "Asia/Seoul",
      "timeFormat": "auto" // 'auto'(OS 설정 따름), '12', '24'
    }
  }
}
```

- `userTimezone`: 프롬프트 컨텍스트에 사용할 사용자 로컬 시간대입니다.
- `timeFormat`: 12시간제 또는 24시간제 표시 방식을 선택합니다.

## 도구 및 커넥터 (원본 시간 유지)

각 채널 도구(Slack, Discord 등)는 서비스 제공자가 제공하는 **원본 타임스탬프**를 그대로 반환하면서도, 일관성을 위해 다음과 같은 필드를 추가합니다:

- `timestampMs`: 유닉스 밀리초 (UTC)
- `timestampUtc`: ISO 8601 형식의 UTC 문자열

따라서 원본 정보가 손실되지 않으며, 모델은 이를 바탕으로 정확한 전송 시각을 파악할 수 있습니다.
