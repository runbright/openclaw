# 그룹 메시지 (WhatsApp 전용)

이 문서는 왓츠앱 웹 채널에서 그룹 메시지를 처리하는 구체적인 방식과 설정에 대해 설명합니다.

## 주요 기능 (왓츠앱 특화)

- **활성화 모드 (`activation`)**:
    - `mention` (기본값): 봇을 직접 태그(`@이름`)하거나 전화번호가 포함된 경우에만 응답합니다.
    - `always`: 모든 메시지를 읽지만, 의미 있는 답변을 할 수 있을 때만 응답합니다. (그룹 채팅창에서 `/activation always` 명령어로 변경 가능)
- **맥락 주입 (Context Injection)**: 봇이 응답하지 않았던 이전 대화 기록들(기본 최근 50개)을 답변 생성 시 참고 자료로 활용합니다. "아까 누가 뭐라고 했지?" 같은 질문에 답할 수 있습니다.
- **발신자 표시**: 에이전트에게 전달되는 모든 메시지 끝에 `[from: 보낸사람 이름 (+번호)]`가 붙어, 에이전트가 누구와 대화 중인지 알 수 있게 합니다.
- **이미지/보기 한 번 메시지**: 왓츠앱의 '보기 한 번' 메시지나 만료되는 메시지도 에이전트가 내용을 파악하고 멘션 여부를 확인할 수 있습니다.

## 설정 예시 (openclaw.json)

에이전트 이름이 "openclaw"인 경우, 아래와 같이 설정하면 `@openclaw` 멘션에 반응합니다.

```json5
{
  "agents": [
    {
      "id": "main",
      "groupChat": {
        "historyLimit": 50,
        "mentionPatterns": ["@?openclaw", "\\+?821012345678"]
      }
    }
  ]
}
```

## 사용 팁

- **개별 세션**: 그룹 대화는 개인 쪽지(DM) 대화와 완전히 분리된 세션을 가집니다. 그룹에서 `/reset`을 해도 개인 대화 기록은 지워지지 않습니다.
- **하트비트 생략**: 그룹 창을 시끄럽게 하지 않기 위해, 그룹 세션에서는 주기적 알림(Heartbeat)을 보내지 않습니다.
- **명령어 사용**: `/verbose on`, `/think high` 같은 명령어를 그룹 창에 단독 메시지로 보내면 해당 그룹 대화에만 설정이 적용됩니다.
