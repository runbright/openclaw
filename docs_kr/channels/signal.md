# 시그널 (Signal / signal-cli)

시그널 채널은 강력한 보안과 개인정보 보호를 중시하는 대화 채널입니다. OpenClaw는 외부 도구인 `signal-cli`를 통해 시그널 네트워크에 접속합니다.

## 준비 사항
- 서버에 `signal-cli`가 설치되어 있어야 합니다.
- 인증용 SMS를 받을 수 있는 전화번호(별도 번호 권장)가 필요합니다.

## 빠른 설정 방법

시그널을 연결하는 방법은 크게 두 가지입니다.

### 방법 A: 기존 계정 연결 (QR 코드)
이미 사용 중인 시그널 계정을 에이전트로 쓰고 싶을 때 사용합니다.
1.  서버에서 `signal-cli link -n "OpenClaw"`를 실행합니다.
2.  화면에 나오는 QR 코드를 스마트폰의 시그널 앱(설정 > 연결된 기기)으로 스캔합니다.

### 방법 B: 새 번호 등록 (SMS 인증)
에이전트 전용의 새로운 번호를 만들 때 사용합니다.
1.  번호 등록 요청: `signal-cli -a +8210... register`
2.  인증 코드 확인 후 입력: `signal-cli -a +8210... verify <인증코드>`
    - 캡차(Captcha)가 필요한 경우, `signalcaptchas.org`에서 생성한 코드를 함께 입력해야 합니다.

## OpenClaw 설정 (openclaw.json)
```json5
{
  "channels": {
    "signal": {
      "enabled": true,
      "account": "+821012345678", // 등록한 번호
      "cliPath": "signal-cli",
      "dmPolicy": "pairing"
    }
  }
}
```

## 주요 특징
- **강력한 보안**: 모든 대화는 종단간 암호화(E2EE) 처리됩니다.
- **페어링 시스템**: 처음 대화할 때 페어링 코드로 승인해야 대화가 시작됩니다.
- **타이핑 효과**: 에이전트가 답변을 생각하는 동안 채팅창에 '입력 중' 표시가 나타납니다.
- **반응(Reaction)**: 메시지에 이모지 리액션을 남길 수 있는 기능을 지원합니다.

## 문제 해결 팁
- **답장이 없을 때**: `openclaw pairing list signal` 명령어로 승인 대기 중인 사용자가 있는지 확인하세요.
- **데이터 백업**: 시그널 계정 정보는 `~/.local/share/signal-cli/`에 저장됩니다. 서버를 옮길 때는 이 폴더를 반드시 백업해야 합니다.
