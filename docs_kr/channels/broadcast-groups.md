# 브로드캐스트 그룹 (Broadcast Groups)

**상태:** 실험적 기능 (Experimental)

## 개요
브로드캐스트 그룹은 여러 명의 에이전트가 하나의 메시지를 동시에 처리하고 응답할 수 있게 해주는 기능입니다. 이를 통해 왓츠앱 그룹이나 DM 하나에 코드 리뷰어, 보안 전문가, 문서 작성가 등 전문화된 에이전트 팀을 구성할 수 있습니다.

현재는 **왓츠앱(WhatsApp)** 웹 채널에서만 지원됩니다.

## 주요 사용 사례
- **전문가 팀 구성**: 하나의 질문에 대해 코드 리뷰어와 보안 분석가가 각자의 시각에서 동시에 답변합니다.
- **다국어 지원**: 사용자의 질문에 대해 영어 담당 에이전트와 한국어 담당 에이전트가 각각 답변을 제공합니다.
- **업무 자동화**: 메시지 하나로 작업 추적 에이전트와 시간 기록 에이전트가 각자의 DB를 동시에 업데이트합니다.

## 설정 방법 (openclaw.json)

`broadcast` 섹션을 추가하고, 왓츠앱 그룹 ID나 전화번호에 대응하는 에이전트 목록을 작성합니다.

```json5
{
  "broadcast": {
    "strategy": "parallel", // "parallel"(동시 실행) 또는 "sequential"(순차 실행)
    "120363403215116621@g.us": ["alfred", "baerbel"] // 그룹 ID와 에이전트 ID 목록
  }
}
```

## 작동 원리
1.  메시지가 들어오면 지정된 모든 에이전트가 **동시**에 실행됩니다.
2.  각 에이전트는 자신만의 **독립된 세션과 기억**을 가집니다 (다른 에이전트가 무슨 말을 했는지 기본적으로 알지 못합니다).
3.  각 에이전트에게 서로 다른 도구 권한(예: 읽기 전용 vs 쓰기 가능)이나 서로 다른 모델(예: Claude 3.5 vs GPT-4)을 부여할 수 있습니다.

## 주의사항
- **성능**: 너무 많은 에이전트(10명 이상)를 한 그룹에 넣으면 응답 속도가 느려질 수 있습니다.
- **응답 순서**: 동시 실행(`parallel`) 모드에서는 에이전트들의 답변 순서가 일정하지 않을 수 있습니다.
- **비용**: 실행되는 에이전트 수만큼 AI 모델 사용 비용이 발생합니다.
