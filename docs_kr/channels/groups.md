# 그룹 대화 (Groups)

OpenClaw는 왓츠앱, 텔레그램, 디스코드, 슬랙 등 모든 지원 채널에서 일관된 그룹 대화 방식을 제공합니다.

## 기본 원칙
1.  **내 계정 그대로**: OpenClaw는 별도의 봇 계정이 아니라 사용자의 기존 계정을 통해 작동합니다. 따라서 사용자가 속해 있는 그룹은 에이전트도 볼 수 있습니다.
2.  **보안 우선**: 기본적으로 그룹 대화는 차단되어 있거나 멘션(`@봇이름`)이 있을 때만 응답하도록 설정되어 있습니다.

## 주요 설정 조합

| 목표 | 설정 방법 |
| :--- | :--- |
| **모든 그룹 허용, 멘션 시에만 답장** | `groups: { "*": { requireMention: true } }` |
| **모든 그룹 답장 금지** | `groupPolicy: "disabled"` |
| **특정 그룹만 허용** | `groups: { "<그룹ID>": { ... } }` |
| **나만 그룹에서 명령 가능** | `groupPolicy: "allowlist"`, `groupAllowFrom: ["+8210..."]` |

## 작동 흐름
- **계승**: 그룹 대화 정책(`groupPolicy`)이 최우선이며, 그 다음 허용 목록(`allowlist`), 마지막으로 멘션 여부를 확인합니다.
- **멘션(Mention)**: 봇을 직접 태그하거나, 봇의 답변에 답글(Reply)을 다는 것도 멘션으로 간주되어 응답을 유도합니다.

## 세션 및 격리 (Sandboxing)
그룹 대화는 개인 DM과는 별도의 세션 키를 가집니다. 보안을 위해 그룹 대화는 도커(Docker) 샌드박스 안에서만 실행되도록 설정하여, 에이전트가 내 컴퓨터의 중요한 파일을 건드리지 못하게 할 수 있습니다.

- `mode: "non-main"`: 메인 대화(DM)가 아닌 모든 대화(그룹 등)를 샌드박스 환경에서 실행합니다.

## 채널별 특징
- **텔레그램**: 포럼 형식의 그룹에서는 각 주제(Topic)별로 독립된 대화 세션을 가집니다.
- **왓츠앱**: 그룹 ID는 보통 `숫자@g.us` 형식을 보입니다. `openclaw logs`를 통해 확인할 수 있습니다.
- **디스코드/슬랙**: 서버 및 채널 단위로 세밀한 권한 설정이 가능합니다.
